Hoisting & var / let / const Warm-up

Hoisting rules & TDZ

▶

Hoisting is JS's behavior of moving declarations to the top of their scope before execution — so variables and functions can be referenced before their line in code.

JS engine does a two-pass scan: first collects declarations, then executes. Only declarations are hoisted — not initializations.

DeclarationHoisted?Initialized?Usable before?
var xYesYes (undefined)Yes → undefined
let xYesNo (TDZ)No – ReferenceError
const xYesNo (TDZ)No – ReferenceError
function declarationYesYes (full body)Yes ✓
classYesNo (TDZ)No – ReferenceError
console.log(x);  // undefined
var x = 5;

greet();          // works — function fully hoisted
function greet() { console.log("Hello"); }

sayHi();          // TypeError — var hoisted, value = undefined
var sayHi = function() {};

Event Loop Core

Call Stack → Microtasks → Macrotasks

▶

The event loop is the mechanism that allows JS (single-threaded) to handle async operations — it continuously checks if the call stack is empty, then processes queued tasks.

  1. Call Stack — synchronous code runs here
  2. Microtask Queue — Promise.then/catch/finally, queueMicrotask(), MutationObserver
  3. Macrotask Queue — setTimeout, setInterval, UI events, I/O (one per loop tick)
After every single macrotask, the entire microtask queue is drained before the next macrotask.
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1 → 4 → 3 → 2

Complex example + async/await

▶
setTimeout(() => {
  console.log('A');
  Promise.resolve().then(() => console.log('B'));
}, 0);
Promise.resolve()
  .then(() => { console.log('C'); return Promise.resolve(); })
  .then(() => console.log('D'));
console.log('E');
// E → C → D → A → B
async function main() {
  console.log('start');
  await Promise.resolve();
  console.log('after await'); // microtask
}
main();
console.log('sync');

this, Context & Closures Core

5 rules for this + call/apply/bind

▶

The this keyword refers to the object that is currently executing the function — its value is determined by how the function is called, not where it's defined.

ContextValue of this
Global (non-strict)window / global
Global (strict mode)undefined
Object methodThe object before the dot
Constructor (new)The newly created object
Arrow functionLexical — from enclosing scope at definition time

function greet(hi, punct) { return `${hi}, ${this.name}${punct}`; }
const p = { name: 'Bob' };
greet.call(p, 'Hi', '!');       // immediate, args spread
greet.apply(p, ['Hey', '.']);   // immediate, args array
const bound = greet.bind(p, 'Hello'); // returns new fn
bound('?');

Closures — definition & use cases

▶

A closure retains access to outer scope variables even after the outer function returns.

function makeCounter(init = 0) {
  let count = init;
  return { inc: () => ++count, dec: () => --count, val: () => count };
}
const c = makeCounter(10);
c.inc(); // 11
Use cases: module pattern, memoization, partial application, event handlers, data encapsulation, factory functions.
Higher-Order Functions

HOF, currying, memoization, composition

▶

A HOF either takes a function as argument or returns a function.

// Currying: Transform a multi-arg function into a sequence of single-arg functions.
const add = a => b => a + b;
const add5 = add(5);
add5(3); // 8

// Generic curry: Adapts any function — collects args until arity is satisfied, then calls.
const curry = fn => function curried(...args) {
  return args.length >= fn.length ? fn(...args) : (...more) => curried(...args, ...more);
};

// Memoize: Cache return values keyed by serialized args to skip repeated computation.
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const r = fn.apply(this, args);
    cache.set(key, r); return r;
  };
}

// Composition & pipe: compose = right-to-left (f(g(x))), pipe = left-to-right (g then f).
const compose = (...fns) => x => fns.reduceRight((v, f) => f(v), x);
const pipe    = (...fns) => x => fns.reduce((v, f) => f(v), x);

Classes (all about)

Class syntax — full reference

▶

A class is syntactic sugar over prototype-based inheritance — a blueprint for creating objects with shared methods, private fields, and constructor logic. Under the hood, it's still prototype chains.

class Animal {
  // Class field (ES2022)
  #name;                          // private field
  species = 'unknown';           // public field
  static count = 0;              // static field

  constructor(name) {
    this.#name = name;
    Animal.count++;
  }

  // Getter / setter
  get name() { return this.#name; }
  set name(v) { this.#name = v; }

  speak() { return `${this.#name} makes noise`; }
  static getCount() { return Animal.count; }
}

class Dog extends Animal {
  #tricks = [];

  constructor(name) {
    super(name);           // must call super first
  }

  speak() {
    return super.speak() + ' — Woof!';
  }

  learn(trick) { this.#tricks.push(trick); }
}

// instanceof works through prototype chain
new Dog('Rex') instanceof Animal; // true

Promises & async/await

Promise states, chaining, error handling

▶

A Promise is an object representing eventual completion or failure of an async operation. States: pending → fulfilled | rejected (immutable once settled).

// Create: Wrap an async operation — call resolve on success, reject on failure.
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('done'), 1000);
});

// Chain — each .then returns a new Promise
p.then(v => v.toUpperCase())
 .then(v => console.log(v))
 .catch(err => console.error(err))
 .finally(() => console.log('done'));

// Static methods: Combinators for coordinating multiple Promises — differ in how they handle rejections.
Promise.all([p1, p2]);          // fail-fast: rejects if any rejects
Promise.allSettled([p1, p2]);   // waits all, returns {status, value/reason}[]
Promise.any([p1, p2]);          // first fulfilled
Promise.race([p1, p2]);         // first settled (fulfilled OR rejected)
Promise.resolve(val);
Promise.reject(err);

async/await — patterns & error handling

▶
// Basic: async/await makes async code look synchronous — always wrap in try/catch.
async function fetchUser(id) {
  try {
    const res = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return await res.json();
  } catch (err) {
    console.error(err);
    throw err;  // re-throw or return fallback
  }
}

// Parallel — don't await one by one!
const [users, posts] = await Promise.all([
  fetch('/api/users').then(r => r.json()),
  fetch('/api/posts').then(r => r.json())
]);

// Top-level await (ES2022, in modules)
const config = await loadConfig();
Never await inside a forEach — it doesn't work as expected. Use for...of for sequential or Promise.all(arr.map(...)) for parallel.
Generators & Iterators

Generator functions

▶

A generator function returns a Generator object. Execution pauses at each yield and resumes on .next().

function* counter(start = 0) {
  while (true) {
    const reset = yield start;  // can receive value via .next(val)
    start = reset ?? start + 1;
  }
}
const gen = counter(1);
gen.next();         // { value: 1, done: false }
gen.next();         // { value: 2, done: false }
gen.next(10);       // { value: 10, done: false } — reset

// Finite generator
function* range(start, end, step = 1) {
  for (let i = start; i < end; i += step) yield i;
}
[...range(0, 10, 2)]; // [0, 2, 4, 6, 8]

Custom iterators & async generators

▶
// Custom iterable object
const iterable = {
  [Symbol.iterator]() {
    let n = 0;
    return { next: () => n < 3
      ? { value: n++, done: false }
      : { value: undefined, done: true } };
  }
};
[...iterable]; // [0, 1, 2]

// Async generator — paginate API
async function* paginate(url) {
  let page = 1;
  while (true) {
    const data = await fetch(`${url}?page=${page++}`).then(r => r.json());
    if (!data.length) return;
    yield data;
  }
}
for await (const page of paginate('/api/items')) {
  process(page);
}
