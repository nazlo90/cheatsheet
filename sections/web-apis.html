Storage APIs

localStorage vs sessionStorage vs Cookies vs IndexedDB

▶

Browser storage APIs let you persist data client-side. The right choice depends on data size, required lifetime, and whether it needs to be sent to the server.

Never store auth tokens in localStorage — XSS vulnerable. Use httpOnly cookies for sensitive auth data.

WebSocket API & lifecycle

▶
// Connection lifecycle
const ws = new WebSocket('wss://api.example.com/ws');

ws.onopen    = () => ws.send(JSON.stringify({ type: 'auth', token }));
ws.onmessage = (e) => { const data = JSON.parse(e.data); handle(data); };
ws.onclose   = (e) => console.log(`Closed: ${e.code} ${e.reason}`);
ws.onerror   = (e) => console.error('WS Error', e);

ws.send('text');
ws.send(JSON.stringify({ type: 'message', body: 'hello' }));
ws.close(1000, 'Normal closure');

// ReadyState
// CONNECTING=0, OPEN=1, CLOSING=2, CLOSED=3
ws.readyState === WebSocket.OPEN;

Auto-reconnect pattern + RxJS WebSocket

▶
// Reconnect with exponential backoff
class ReconnectingWS {
  private ws: WebSocket | null = null;
  private retries = 0;

  connect() {
    this.ws = new WebSocket(this.url);
    this.ws.onclose = () => {
      const delay = Math.min(1000 * 2 ** this.retries++, 30000);
      setTimeout(() => this.connect(), delay);
    };
  }
}

// RxJS webSocket
import { webSocket } from 'rxjs/webSocket';
const subject = webSocket('wss://api.example.com/ws');
subject.subscribe(msg => handle(msg));
subject.next({ type: 'subscribe', channel: 'prices' });
subject.pipe(
  retryWhen(errors => errors.pipe(delay(2000)))
).subscribe();
In Angular, always close WebSocket on ngOnDestroy. Use shareReplay(1) to share a single connection across multiple components.

Service Workers

Concept, lifecycle & caching strategies

▶

Separate thread acting as network proxy. HTTPS only. Enables offline, background sync, push notifications.

Lifecycle: Installing → Waiting → Activating → Activated → Redundant

navigator.serviceWorker.register('/sw.js');
self.addEventListener('install', e => e.waitUntil(caches.open('v1').then(c => c.addAll(['/']))));
self.addEventListener('fetch', e => e.respondWith(
  caches.match(e.request).then(cached => cached || fetch(e.request))
));
StrategyUse when
Cache firstStatic assets
Network firstAPI data — freshness critical
Stale-while-revalidateBalance: serve cache, refresh in bg
Network onlyAuth requests
Web Workers

Web Workers vs Service Workers

▶

Web Workers run scripts in background threads, separate from the main UI thread — keeping the page responsive during heavy CPU work. No DOM access; communicate via postMessage.

Web WorkerService Worker
PurposeCPU computationNetwork proxy / offline
LifecycleTied to pageIndependent
DOMNo accessNo access
const w = new Worker('worker.js');
w.postMessage({ data: largeArr });
w.onmessage = e => console.log(e.data);
// worker.js
self.onmessage = e => self.postMessage(heavyWork(e.data.data));
Angular 18 added official Web Worker support. Use for: sorting/filtering, encryption, image processing — anything blocking 16ms+.
