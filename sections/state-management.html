When does global state become a liability? When every small UI tweak (a tooltip open/close, tab selection) goes through actions and reducers — that's a sign to use component-local state (ComponentStore or Signals) instead.

ComponentStore — local, reactive state

▶
// ComponentStore: local state for a smart component — no global store needed
import { ComponentStore } from '@ngrx/component-store';

interface TodoState { todos: Todo[]; loading: boolean; }

@Injectable()
class TodoStore extends ComponentStore<TodoState> {
  constructor() { super({ todos: [], loading: false }); }

  // Selector — slice of state as Observable
  readonly todos$ = this.select(s => s.todos);
  readonly loading$ = this.select(s => s.loading);

  // Updater — synchronous state mutation
  readonly addTodo = this.updater((state, todo: Todo) => ({
    ...state, todos: [...state.todos, todo]
  }));

  // Effect — handles async side-effects
  readonly loadTodos = this.effect<void>($ =>
    $.pipe(
      tap(() => this.patchState({ loading: true })),
      switchMap(() => this.http.get<Todo[]>('/api/todos').pipe(
        tap(todos => this.patchState({ todos, loading: false }))
      ))
    )
  );
}

// Usage: provide in component — destroyed with component
@Component({ providers: [TodoStore] })
class TodoComponent {
  readonly store = inject(TodoStore);
  todos$ = this.store.todos$;
}
NgRx Full Guide State

Core concepts + full example

▶

Store → Selector → Component → dispatch(Action) → Reducer/Effect → Store

// Actions: Plain objects describing what happened. They have a type and optional payload.
const loadUsers = createAction('[Users] Load');
const loadSuccess = createAction('[Users] Success', props<{ users: User[] }>());
const loadFailure = createAction('[Users] Failure', props<{ error: string }>());

// Reducers: Pure functions that take the current state and an action, and return a new state.
const reducer = createReducer(initialState,
  on(loadUsers, s => ({ ...s, loading: true })),
  on(loadSuccess, (s, { users }) => ({ ...s, users, loading: false })),
  on(loadFailure, (s, { error }) => ({ ...s, error, loading: false }))
);

// Selectors: Functions that select (extract) slices of state from the store.
const selectState  = createFeatureSelector<UsersState>('users');
const selectAll    = createSelector(selectState, s => s.users);
const selectById   = (id: string) => createSelector(selectAll, us => us.find(u => u.id === id));

// Effects: Handle side effects (like HTTP calls) by listening to actions and dispatching new actions asynchronously.
loadUsers$ = createEffect(() => this.actions$.pipe(
  ofType(loadUsers),
  switchMap(() => this.svc.getAll().pipe(
    map(users => loadSuccess({ users })),
    catchError(err => of(loadFailure({ error: err.message })))
  ))
));

NgRx tips & when NOT to use it

▶
  • Actions describe events, not commands — [Users Page] Load Requested
  • Reducers must be pure and immutable — never mutate state
  • Use selectors — decouple components from state shape + memoization
  • Use DevTools for time-travel debugging
Don't use NgRx for: simple feature-local state, single-component state, straightforward forms. Use BehaviorSubject services or Signals instead.
NgRx SignalStore Angular 17+

What is SignalStore & when to use it

▶

SignalStore (@ngrx/signals) is a lightweight, Signal-based state management solution introduced in NgRx 17. It replaces the Redux-style ceremony (actions/reducers/effects) for feature-local or mid-complexity state that doesn't need global NgRx Store.

NgRx StoreNgRx SignalStore
API styleActions → Reducers → EffectsSignals + methods directly on the store
BoilerplateHighLow
ScopeGlobal / featureFeature-local or global
ReactivityRxJS ObservablesAngular Signals (+ optional RxJS via rxMethod)
DevToolsFull NgRx DevToolsBasic (state snapshot support)
Best forLarge app-wide shared stateFeature state, UI state, smaller apps
Use SignalStore for feature-scoped state where NgRx Store feels heavy. Use NgRx Store when you need time-travel debugging, complex cross-feature state, or large team conventions.

Basic store — full example

▶
import { signalStore, withState, withComputed, withMethods, withHooks, patchState } from '@ngrx/signals';
import { rxMethod } from '@ngrx/signals/rxjs-interop';
import { computed, inject } from '@angular/core';
import { switchMap, tap } from 'rxjs/operators';

interface UsersState {
  users: User[];
  loading: boolean;
  error: string | null;
}

export const UsersStore = signalStore(
  { providedIn: 'root' },          // or provide in component for local scope

  withState<UsersState>({           // initial state
    users: [],
    loading: false,
    error: null,
  }),

  withComputed(({ users }) => ({    // derived Signals
    totalUsers: computed(() => users().length),
    activeUsers: computed(() => users().filter(u => u.active)),
  })),

  withMethods((store, http = inject(HttpClient)) => ({
    // Sync update
    clearError() {
      patchState(store, { error: null });
    },
    // Async via rxMethod (replaces Effects)
    loadUsers: rxMethod<void>(
      pipe(
        tap(() => patchState(store, { loading: true, error: null })),
        switchMap(() => http.get<User[]>('/api/users')),
        tap({
          next: users => patchState(store, { users, loading: false }),
          error: err  => patchState(store, { error: err.message, loading: false }),
        })
      )
    ),
  })),

  withHooks({
    onInit(store) { store.loadUsers(); },  // auto-load on creation
  })
);

Using the store in a component

▶
// Inject and use — no selectors, no dispatch, just Signals
@Component({
  template: `
    @if (store.loading()) { <p>Loading…</p> }
    @for (user of store.activeUsers(); track user.id) {
      <li>{{ user.name }}</li>
    }
    <p>Total: {{ store.totalUsers() }}</p>
  `,
  providers: [UsersStore],  // omit if providedIn:'root'
  changeDetection: ChangeDetectionStrategy.OnPush,
})
export class UsersComponent {
  readonly store = inject(UsersStore);
}
Each property in withState() auto-becomes a Signal on the store instance — read with store.users(), update only via patchState() to keep mutations traceable.

withEntities — built-in CRUD

▶
import { withEntities, setAllEntities, addEntity, updateEntity, removeEntity } from '@ngrx/signals/entities';

export const ProductsStore = signalStore(
  withEntities<Product>(),   // adds: ids[], entities{}, entityMap

  withMethods((store, http = inject(HttpClient)) => ({
    load: rxMethod<void>(
      pipe(
        switchMap(() => http.get<Product[]>('/api/products')),
        tap(products => patchState(store, setAllEntities(products)))
      )
    ),
    add:    (p: Product) => patchState(store, addEntity(p)),
    update: (id: string, changes: Partial<Product>) =>
              patchState(store, updateEntity({ id, changes })),
    remove: (id: string) => patchState(store, removeEntity(id)),
  }))
);

Custom store features (reusable mixins)

▶

Extract cross-cutting concerns into reusable store features — the SignalStore equivalent of NgRx meta-reducers.

// Reusable loading/error feature
function withLoadingState() {
  return signalStoreFeature(
    withState({ loading: false, error: null as string | null }),
    withMethods(store => ({
      setLoading: () => patchState(store, { loading: true,  error: null }),
      setError:  (error: string) => patchState(store, { loading: false, error }),
      setLoaded: () => patchState(store, { loading: false }),
    }))
  );
}

// Compose into any store
export const OrdersStore = signalStore(
  withState({ orders: [] as Order[] }),
  withLoadingState(),   // injects loading + error + helper methods
  withMethods(store => ({ /* … */ }))
);
