AngularJS vs Angular 2+ & key building blocks

▶

Angular is a TypeScript-based framework for building SPAs — component-based architecture with built-in DI, routing, reactive forms, and a powerful CLI.

AngularJS (1.x)Angular (2+)
LanguageJavaScriptTypeScript
ArchitectureMVCMVVM (Component + Template + Service)
Building blockDirectivesComponents
Change detectionDigest cycleZone.js + OnPush + Signals

  • Directive — modify DOM. Structural (@if, @for) or Attribute

Angular Directives Angular

What is a directive & types

▶

A directive is a class decorated with @Directive that adds behaviour or structure to DOM elements. Components are a special kind of directive with a template.

TypePurposeExamples
ComponentDirective + template + styles@Component — every component
StructuralChanges DOM structure (adds/removes elements)*ngIf, *ngFor, *ngSwitch, @if, @for
AttributeChanges appearance or behaviour of host elementNgClass, NgStyle, RouterLink, custom
// Attribute directive — highlight on hover
@Directive({
  selector: '[appHighlight]',
  standalone: true
})
export class HighlightDirective {
  @Input('appHighlight') color = 'yellow';

  private el = inject(ElementRef);

  @HostListener('mouseenter') onEnter() {
    this.el.nativeElement.style.backgroundColor = this.color;
  }
  @HostListener('mouseleave') onLeave() {
    this.el.nativeElement.style.backgroundColor = '';
  }
}

<!-- <p [appHighlight]="'lightblue'">Hover me</p> -->

Structural directives — microsyntax & custom

▶

Structural directives use * microsyntax sugar — *ngIf="x" desugars to [ngIf]="x" on a hidden <ng-template>. Custom structural directives inject TemplateRef and ViewContainerRef.

// Custom structural directive — *appUnless (opposite of *ngIf)
@Directive({ selector: '[appUnless]', standalone: true })
export class UnlessDirective {
  private vcr = inject(ViewContainerRef);
  private tmpl = inject(TemplateRef);
  private hasView = false;

  @Input() set appUnless(condition: boolean) {
    if (!condition && !this.hasView) {
      this.vcr.createEmbeddedView(this.tmpl);
      this.hasView = true;
    } else if (condition && this.hasView) {
      this.vcr.clear();
      this.hasView = false;
    }
  }
}

<!-- <div *appUnless="isLoggedIn">Please log in</div> -->

// Modern built-in equivalents (Angular 17+ template syntax)
// @if (show) { <p>visible</p> } @else { <p>hidden</p> }
// @for (item of items; track item.id) { <li>{{ item.name }}</li> }
// @switch (status) { @case ('active') { <span>Active</span> } }
Prefer modern @if/@for/@switch (Angular 17+) over *ngIf/*ngFor — they compile more efficiently (no extra DOM wrapper) and have cleaner syntax.

HostBinding, HostListener & exportAs

▶
// @HostBinding — bind directive property to host element attribute/class/style
@Directive({ selector: '[appActive]', standalone: true })
export class ActiveDirective {
  @Input() appActive = false;

  @HostBinding('class.active') get isActive() { return this.appActive; }
  @HostBinding('attr.aria-selected') get ariaSelected() { return this.appActive; }
  @HostBinding('style.fontWeight') get fw() { return this.appActive ? 'bold' : 'normal'; }
}

// exportAs — expose directive instance to template variable
@Directive({ selector: '[appTooltip]', exportAs: 'tooltip', standalone: true })
export class TooltipDirective {
  visible = false;
  show() { this.visible = true; }
  hide() { this.visible = false; }
}

Angular Pipes Angular

What is a pipe & built-in pipes

▶

A pipe is a template transformation function — it takes an input value and optional arguments and returns a transformed value. Used with the | operator in templates.

PipeExampleOutput
date{{ today | date:'dd/MM/yyyy' }}20/04/2026
currency{{ 1234.5 | currency:'USD' }}$1,234.50
number{{ 3.14159 | number:'1.2-2' }}3.14
percent{{ 0.25 | percent }}25%
uppercase / lowercase{{ 'hello' | uppercase }}HELLO
titlecase{{ 'hello world' | titlecase }}Hello World
json{{ obj | json }}pretty-printed JSON
slice{{ items | slice:0:3 }}first 3 items
keyvalue*ngFor="let p of obj | keyvalue"key/value pairs
async{{ users$ | async }}unwrapped Observable/Promise

Pure vs impure pipes & custom pipe

▶
Pure pipe (default)Impure pipe
Re-runs whenInput reference changesEvery change detection cycle
PerformanceMemoized — skipped if input unchanged ✓Called constantly ✗
Use forFormatting, sorting stable dataFiltering mutable arrays, async pipe
Declarationpure: true (default)pure: false
// Custom pure pipe — format file size
@Pipe({ name: 'fileSize', standalone: true })
export class FileSizePipe implements PipeTransform {
  transform(bytes: number, decimals = 2): string {
    if (!bytes) return '0 B';
    const k = 1024;
    const units = ['B', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return `${(bytes / k ** i).toFixed(decimals)} ${units[i]}`;
  }
}
<!-- {{ 1048576 | fileSize }} → '1.00 MB' -->

// Custom impure pipe — filter mutable array
@Pipe({ name: 'filterBy', standalone: true, pure: false })
export class FilterByPipe implements PipeTransform {
  transform<T>(items: T[], key: keyof T, value: unknown): T[] {
    return items.filter(item => item[key] === value);
  }
}
<!-- {{ users | filterBy:'role':'admin' }} -->

// Pipe with DI — inject services inside pipes
@Pipe({ name: 'translate', standalone: true })
export class TranslatePipe implements PipeTransform {
  private i18n = inject(I18nService);
  transform(key: string): string { return this.i18n.get(key); }
}
Avoid impure pipes on large lists — they run on every CD cycle. Prefer pure pipes + immutable data, or use a computed() signal for reactive filtering.
Dependency Injection Angular

DI system — hierarchical injectors

▶

Angular has a hierarchical DI system. Each component has its own injector; if a dependency is not found, it walks up to the parent injector, then to the root injector.

Injector hierarchy: Platform → Root (AppModule) → Feature Module → Component → Element

// Singleton in root
@Injectable({ providedIn: 'root' })   // tree-shakeable
class AuthService {}

// Scoped to component (new instance per component)
@Component({ providers: [MyService] })
class MyComponent {}

// InjectionToken — for non-class values
const API_URL = new InjectionToken<string>('API_URL');
// providers: [{ provide: API_URL, useValue: 'https://...' }]
inject(API_URL); // functional injection (Angular 14+)

// @Optional — null if not found
constructor(@Optional() private logger?: LoggerService) {}

// @Self — only look in own injector
constructor(@Self() private svc: MyService) {}

// @SkipSelf — skip own, look in parent
constructor(@SkipSelf() private parent: MyService) {}

// @Host — stop at host element boundary
constructor(@Host() private svc: MyService) {}

// Factory provider — use when creation needs logic or async init
{ provide: Logger, useFactory: (http: HttpClient) => new RemoteLogger(http), deps: [HttpClient] }

// Multi-provider (interceptors, validators)
{ provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }

Factory provider — patterns & async init

▶

useFactory gives full control over instance creation — accepts injected deps, can contain conditional logic, and is the only provider type that supports async initialization via APP_INITIALIZER.

// 1. Conditional factory — pick implementation based on environment
const loggerProvider = {
  provide: Logger,
  useFactory: (env: Environment) =>
    env.production ? new RemoteLogger() : new ConsoleLogger(),
  deps: [Environment]
};

// 2. Async factory via APP_INITIALIZER — blocks app bootstrap until resolved
export function initializeApp(configService: ConfigService) {
  return () => configService.load();  // returns Promise or Observable
}

providers: [
  ConfigService,
  {
    provide: APP_INITIALIZER,
    useFactory: initializeApp,
    deps: [ConfigService],
    multi: true
  }
]

// 3. InjectionToken factory (Angular 14+ functional style)
const BASE_URL = new InjectionToken<string>('BASE_URL', {
  providedIn: 'root',
  factory: () => inject(DOCUMENT).location.origin
});
All deps in the deps array must be registered in the same or parent injector. For optional deps use new Optional() in the deps array: deps: [HttpClient, new Optional(), SomeOptionalDep]

Multi-providers — collecting multiple implementations

▶

multi: true tells Angular to accumulate all providers for a token into an array instead of replacing the previous one. The injected value is T[].

// Built-in multi-provider tokens
// HTTP_INTERCEPTORS   → HttpInterceptor[]
// APP_INITIALIZER     → (() => Promise|Observable)[]
// NG_VALIDATORS       → Validator[]

// Custom plugin token
interface DashboardWidget { render(): string; }
const DASHBOARD_WIDGETS = new InjectionToken<DashboardWidget[]>('DASHBOARD_WIDGETS');

providers: [
  { provide: DASHBOARD_WIDGETS, useClass: SalesWidget,   multi: true },
  { provide: DASHBOARD_WIDGETS, useClass: UsersWidget,   multi: true },
  { provide: DASHBOARD_WIDGETS, useClass: RevenueWidget, multi: true },
]

// Inject as array
class DashboardComponent {
  widgets = inject(DASHBOARD_WIDGETS);  // DashboardWidget[]
}

// Custom NG_VALIDATORS multi-provider
@Directive({ selector: '[appForbiddenName]', providers: [
  { provide: NG_VALIDATORS, useExisting: ForbiddenNameDirective, multi: true }
]})
class ForbiddenNameDirective implements Validator {
  validate(control: AbstractControl): ValidationErrors | null { /* ... */ }
}
Use useExisting (not useClass) when registering a directive as a validator — useClass creates a second instance; useExisting reuses the one Angular already created.

Change Detection Angular

Default vs OnPush + Zone.js

▶

Zone.js monkey-patches all async APIs and notifies Angular after async work. Default: checks whole tree. OnPush: skips subtree unless:

  • An @Input() reference changes (not mutation)
  • An event fires inside component
  • Observable emits via async pipe
  • markForCheck() / detectChanges() called
@Component({ changeDetection: ChangeDetectionStrategy.OnPush })
class CardComponent {
  @Input() data!: Data;   // must pass NEW reference

  constructor(private cd: ChangeDetectorRef) {}

  update() {
    this.cd.markForCheck();    // schedule check in next cycle
    // OR
    this.cd.detectChanges();   // immediate synchronous check
  }
  pauseCD()  { this.cd.detach(); }
  resumeCD() { this.cd.reattach(); }
Change Detection: RxJS vs Signals Interview

How CD is triggered — two paradigms

▶

The core question: how does Angular know something changed and which views to update?

RxJS + Zone.jsSignals (Angular 16+)
CD triggerZone.js patches async APIs → notifies Angular after every taskSignal mutation → Angular marks only views that read that signal
GranularityComponent subtree (coarse)Template expression (fine-grained)
Zone.js requiredYesNo (zoneless supported)
Template syntax{{ users$ | async }}{{ users() }} — call as function
How async pipe worksSubscribes + calls markForCheck() on emitN/A — toSignal(obs$) bridges interop
Manual triggermarkForCheck() / detectChanges()No manual trigger — reactive automatically
Performance ceilingOnPush reduces checks, still component-levelOnly the exact binding re-evaluates

Code comparison — same feature, two approaches

▶
// ─── RxJS / Zone.js approach ───────────────────────────────
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    <div *ngFor="let u of users$ | async">{{ u.name }}</div>
    <span>{{ count$ | async }}</span>
  `
})
class UserListComponent {
  users$ = this.http.get<User[]>('/api/users').pipe(shareReplay(1));
  count$ = this.users$.pipe(map(u => u.length));
  // async pipe: subscribes + calls markForCheck() on each emit
  // Zone.js: fires CD after HTTP, setTimeout, events...
  // with OnPush: only checks THIS subtree, not whole tree
}

// ─── Signals approach ──────────────────────────────────────
@Component({
  template: `
    <div *ngFor="let u of users()">{{ u.name }}</div>
    <span>{{ count() }}</span>
  `
})
class UserListComponent {
  users = signal<User[]>([]);
  count = computed(() => this.users().length);  // auto-tracks users

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<User[]>('/api/users').subscribe(u => this.users.set(u));
    // OR: users = toSignal(this.http.get(...), { initialValue: [] });
  }
  // Angular tracks which template expressions read users()
  // On users.set() → only those bindings re-evaluate, nothing else
  // No Zone.js, no markForCheck() needed
}

RxJS interop — toSignal / toObservable

▶
import { toSignal, toObservable } from '@angular/core/rxjs-interop';

// Observable → Signal (subscribes automatically, completes on destroy)
class SearchComponent {
  query = signal('');

  // Signal → Observable → apply RxJS operators → back to Signal
  results = toSignal(
    toObservable(this.query).pipe(
      debounceTime(300),
      distinctUntilChanged(),
      switchMap(q => this.http.get(`/search?q=${q}`))
    ),
    { initialValue: [] }
  );
  // template: <li *ngFor="let r of results()">{{ r.name }}</li>
  // Best of both worlds: RxJS operators + signals in template
}

// Rule of thumb:
// - Use RxJS for complex async flows (debounce, retry, combine streams)
// - Use Signals for UI state, derived values, template bindings
// - Bridge with toSignal() at the boundary
Migration path: keep RxJS for business logic, wrap with toSignal() at the component boundary. Over time, replace simple BehaviorSubject state with signals.

signal / computed / effect

▶

A Signal is a reactive primitive that holds a value and notifies consumers automatically when it changes — enabling fine-grained change detection without Zone.js.

import { signal, computed, effect } from '@angular/core';

// Writable signal
const count = signal(0);
count.set(5);
count.update(v => v + 1);

// Computed — auto-tracks dependencies
const doubled = computed(() => count() * 2);

// Effect — runs side effects when deps change
effect(() => console.log('count changed:', count()));

// In template
// <p>{{ count() }}</p>  <!-- call as function -->

// Signal inputs (Angular 17.1+)
import { input, output, model } from '@angular/core';
class MyComponent {
  name = input<string>('');        // replaces @Input()
  nameChange = output<string>();  // replaces @Output()
  value = model<string>('');      // two-way binding
}

▶
import { toSignal, toObservable } from '@angular/core/rxjs-interop';

// Observable → Signal

// Signal → Observable
const count$ = toObservable(count);

// Zoneless (Angular 18+ experimental)
// bootstrapApplication(AppComponent, {
//   providers: [provideExperimentalZonelessChangeDetection()]

Reactive Forms — full example

▶
import { FormBuilder, Validators, FormGroup, FormArray } from '@angular/forms';

class ProfileComponent {
  private fb = inject(FormBuilder);

  form = this.fb.group({
    name:  ['', [Validators.required, Validators.minLength(2)]],
    email: ['', [Validators.required, Validators.email]],
    age:   [null, [Validators.min(18), Validators.max(99)]],
    tags:  this.fb.array([]),   // FormArray
    address: this.fb.group({    // nested FormGroup
      city: ['', Validators.required]
    })
  });

  get tags() { return this.form.get('tags') as FormArray; }
  addTag() { this.tags.push(this.fb.control('')); }

  ngOnInit() {
    // React to value changes
    this.form.get('email')!.valueChanges.pipe(
      debounceTime(300),
      distinctUntilChanged(),
      switchMap(email => this.api.checkEmail(email))
    ).subscribe(taken => { ... });

    // Programmatic updates
    this.form.patchValue({ name: 'Alice' });  // partial
    this.form.setValue({ name: 'Alice', email: 'alice@example.com', age: 25, tags: [], address: { city: 'Kyiv' } }); // full
    this.form.reset();
  }

  submit() {
    if (this.form.invalid) return;
    console.log(this.form.getRawValue());
  }
}

Custom validators & async validators

▶
// Sync custom validator
function noSpacesValidator(control: AbstractControl): ValidationErrors | null {
  return control.value?.includes(' ')
    ? { noSpaces: true }
    : null;
}

// Async validator (check unique email)
function emailTakenValidator(api: ApiService): AsyncValidatorFn {
  return (control) => of(control.value).pipe(
    debounceTime(300),
    switchMap(v => api.emailExists(v)),
    map(exists => exists ? { emailTaken: true } : null)
  );
}

// Cross-field validator (group level)
function passwordMatch(group: AbstractControl): ValidationErrors | null {
  const pwd = group.get('password')?.value;
  const confirm = group.get('confirmPassword')?.value;
  return pwd === confirm ? null : { mismatch: true };
}

// Usage
fb.group({ password: [''], confirmPassword: [''] }, { validators: passwordMatch })

@defer — deferred loading (Angular 17+)

▶

@defer lazily loads a component/block and its dependencies — compiler-driven, not runtime. The chunk is only downloaded when the trigger fires.

TriggerWhen it firesTypical use
on idleBrowser is idle (requestIdleCallback)Below-fold content, analytics widgets
on viewportPlaceholder enters the viewport (IntersectionObserver)Charts, images, infinite-scroll items
on interactionUser clicks or touches the placeholderRich editors, dialogs opened on click
on hoverUser hovers over the placeholderTooltips, preview panels
on immediateAs soon as possible after renderNon-critical but above-fold content
on timer(Xms)After a fixed delaySplash screens, delayed banners
when (expr)When the boolean expression becomes trueFeature flags, auth gates, data-driven
// Full @defer anatomy
@defer (on viewport; prefetch on idle) {
  <heavy-chart [data]="data" />
} @loading (minimum 150ms) {
  <skeleton-loader />
} @placeholder (minimum 500ms) {
  <div class="chart-placeholder">Chart loading soon</div>
} @error {
  <p>Failed to load. <button (click)="retry()">Retry</button></p>
}

// when() — data-driven trigger
@defer (when userLoaded) {
  <user-profile [user]="user" />
}

// Trigger on named element reference
<button #loadBtn>Load Comments</button>
@defer (on interaction(loadBtn)) {
  <comments-section />
}

// Prefetch trigger can differ from render trigger
@defer (on interaction; prefetch on hover) {
  <rich-editor />
}
@placeholder is shown before the trigger fires. @loading is shown while the chunk downloads. Use minimum to prevent flash — Angular shows it for at least that duration even if the chunk loads faster.

Incremental DOM — how Ivy updates the DOM

▶

Ivy uses an incremental DOM approach: instructions are emitted per-component and walk the DOM in-place rather than creating a virtual DOM tree. This means:

  • Memory — no full virtual DOM snapshot in memory; only allocates what changes
  • Tree-shaking — unused rendering instructions (e.g., ɵɵlistener, ɵɵtext) are dropped from the bundle
  • Bundle size — small apps see biggest gains; runtime scales with used features, not framework size
  • SSR — same component code renders on server via platform-server; provideClientHydration() reuses server DOM without full re-render
// Ivy-compiled component emits instruction sequences like:
// ɵɵelementStart(0, 'div')  ɵɵtext(1)  ɵɵelementEnd()
// ɵɵadvance(1); ɵɵtextInterpolate(ctx.title)
// These run IN the existing DOM — no diffing algorithm needed

// Non-destructive hydration (Angular 17+)
bootstrapApplication(AppComponent, {
  providers: [provideClientHydration()]  // reuses SSR DOM, no flicker
});
Ivy's locality enables Angular libraries to ship pre-compiled. strictTemplates: true in tsconfig catches template type errors at build time — use it.

ViewChild, ContentChild & ViewContainerRef Angular

ViewChild vs ContentChild vs ViewContainerRef

▶
ViewChildContentChildViewContainerRef
What it queriesElements in own templateProjected content (ng-content)Anchor for dynamic views
Available fromngAfterViewInitngAfterContentInitInjected via DI or @ViewChild
Typical useDOM access, child component APIWrap content, expose projected stateDynamic component/template creation
// ViewChild — access child component or DOM element
@Component({ template: `<canvas #myCanvas></canvas> <app-chart #chart />` })
class DashboardComponent implements AfterViewInit {
  @ViewChild('myCanvas') canvas!: ElementRef<HTMLCanvasElement>;
  @ViewChild(ChartComponent) chart!: ChartComponent;  // by type

  ngAfterViewInit() {
    // canvas and chart are guaranteed to exist here
    this.chart.render(this.data);
  }
}

// ContentChild — access projected content
@Component({
  selector: 'app-panel',
  template: `<div class="panel"><ng-content /></div>`
})
class PanelComponent implements AfterContentInit {
  @ContentChild(PanelHeaderComponent) header?: PanelHeaderComponent;

  ngAfterContentInit() {
    // projected child available here, not in ngAfterViewInit
    if (this.header) this.header.highlight();
  }
}

// ViewContainerRef — create components dynamically
@Component({ template: `<ng-template #anchor></ng-template>` })
class HostComponent {
  @ViewChild('anchor', { read: ViewContainerRef }) vcr!: ViewContainerRef;

  loadDialog() {
    this.vcr.clear();
    const ref = this.vcr.createComponent(DialogComponent);
    ref.instance.title = 'Hello';
    ref.instance.closed.subscribe(() => ref.destroy());
  }
}

// ViewContainerRef injected directly — current host becomes anchor
@Directive({ selector: '[appDynamic]' })
class DynamicDirective {
  private vcr = inject(ViewContainerRef);
  create() { this.vcr.createComponent(TooltipComponent); }
}
ViewChildren / ContentChildren (plural) return a QueryList — subscribe to .changes for dynamic list updates. Use { static: true } on @ViewChild only if you need it in ngOnInit.
Lifecycle Hooks & Route Guards Angular

Lifecycle hooks order

▶
  1. ngOnChanges — @Input changes (before init and on each change)
  2. ngOnInit — initialized (once)
  3. ngDoCheck — every CD cycle
  4. ngAfterContentInit — projected content ready
  5. ngAfterContentChecked
  6. ngAfterViewInit — view + child views ready
  7. ngAfterViewChecked
  8. ngOnDestroy — cleanup
ngAfterContentInit fires before ngAfterViewInit — content must be ready before view.

Monolith vs Micro Frontend Architecture

Comparison — when to choose each

▶
Monolith (Single SPA)Micro Frontend (MFE)
DeploymentOne build, one deployEach team deploys independently
Shared codeEasy — single codebaseComplex — shared libs must be versioned
Team scaleWorks well up to ~10 engineersNeeded when multiple autonomous teams
Bundle sizeOne large bundle (code-splitting helps)Smaller per-MFE chunks; shared deps tricky
UI consistencyEasy — shared component libraryHard — risk of design divergence
Tech flexibilityLocked to one stackEach MFE can use a different framework
Local devRun one appMust run shell + affected remotes
ComplexityLow ✓High — orchestration, contracts, routing

MFEs add real complexity. Don't adopt them because they "scale" — adopt only when independent deployability actually solves a team/org problem. A well-structured monorepo (Nx) often gives 80% of the benefit without the runtime overhead.

Global ErrorHandler + HTTP interceptor logging

▶
// Custom global error handler — catches all unhandled errors
@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  private logger = inject(LoggingService);

  handleError(error: unknown): void {
    const err = error instanceof Error ? error : new Error(String(error));
    this.logger.captureException(err);
    console.error('[GlobalError]', err);
  }
}

providers: [{ provide: ErrorHandler, useClass: GlobalErrorHandler }]

// HTTP interceptor — log every request/response with timing
@Injectable()
export class LoggingInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<unknown>, next: HttpHandler): Observable<HttpEvent<unknown>> {
    const start = Date.now();
    return next.handle(req).pipe(
      tap(event => {
        if (event instanceof HttpResponse)
          console.log(`[${req.method}] ${req.url} → ${event.status} (${Date.now() - start}ms)`);
      }),
      catchError(err => {
        console.error(`[${req.method}] ${req.url} FAILED`, err);
        return throwError(() => err);
      })
    );
  }
}

Web Vitals & Core Web Vitals thresholds

▶
// Measure Core Web Vitals in Angular app
import { onCLS, onLCP, onINP, onFCP, onTTFB } from 'web-vitals';

function sendToAnalytics({ name, value, id }: Metric) {
  navigator.sendBeacon('/analytics', JSON.stringify({ name, value, id }));
}

onCLS(sendToAnalytics);   // Cumulative Layout Shift
onLCP(sendToAnalytics);   // Largest Contentful Paint
onINP(sendToAnalytics);   // Interaction to Next Paint
onFCP(sendToAnalytics);   // First Contentful Paint
onTTFB(sendToAnalytics);  // Time to First Byte

// Track navigation timing per route
@Injectable({ providedIn: 'root' })
export class RouteTimingService {
  constructor(private router: Router) {
    let navStart = 0;
    router.events.pipe(
      filter(e => e instanceof NavigationStart || e instanceof NavigationEnd)
    ).subscribe(e => {
      if (e instanceof NavigationStart) navStart = Date.now();
      if (e instanceof NavigationEnd)
        console.log(`Route ${e.url} loaded in ${Date.now() - navStart}ms`);
    });
  }
}
