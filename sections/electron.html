Electron Architecture Core

Main Process vs Renderer Process

▶

Electron runs two types of processes. The Main process is Node.js — it controls the app lifecycle and native OS access. Each Renderer process is a Chromium web page — it runs UI code and is sandboxed by default.

Main ProcessRenderer Process
RuntimeNode.jsChromium (browser)
CountOne per appOne per BrowserWindow
Node accessFullNone (sandboxed)
DOM accessNoneFull
Entry filemain.jsindex.html
// main.js — Main Process
const { app, BrowserWindow } = require('electron');

app.whenReady().then(() => {
  const win = new BrowserWindow({
    width: 1200, height: 800,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,   // required for security
      sandbox: true,
    },
  });
  win.loadFile('index.html');
});
Never set nodeIntegration: true in the renderer — it exposes the full Node API to potentially untrusted web content.

Preload script & contextBridge

▶

The preload script runs in the renderer before any web content, with access to a limited set of Node/Electron APIs. contextBridge safely exposes a controlled API to the renderer.

// preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('electronAPI', {
  openFile: () => ipcRenderer.invoke('dialog:openFile'),
  onUpdate: (cb) => ipcRenderer.on('update-available', cb),
  removeUpdate: () => ipcRenderer.removeAllListeners('update-available'),
});

// renderer — window.electronAPI is now available
const filePath = await window.electronAPI.openFile();
Never forward the raw ipcRenderer object through contextBridge — that lets the renderer send arbitrary IPC channels. Expose only named, specific functions.
IPC Communication IPC

ipcMain / ipcRenderer patterns

▶

Inter-process communication (IPC) is the only way for renderer and main processes to exchange data. Use invoke/handle for request-reply and send/on for fire-and-forget.

PatternRenderer sideMain sideUse when
Request / ReplyipcRenderer.invoke()ipcMain.handle()Need a response (async)
One-way (R→M)ipcRenderer.send()ipcMain.on()Fire and forget to main
One-way (M→R)ipcRenderer.on()win.webContents.send()Push from main to renderer
// main.js
ipcMain.handle('dialog:openFile', async () => {
  const { canceled, filePaths } = await dialog.showOpenDialog({});
  if (canceled) return null;
  return filePaths[0];
});

// Push notification from main → renderer
autoUpdater.on('update-available', () => {
  win.webContents.send('update-available');
});
Validate all data received in ipcMain.handle — a compromised renderer can send arbitrary payloads.

IPC channel naming & validation

▶
// Use a whitelist of allowed channels in preload
const ALLOWED_CHANNELS = ['app:getVersion', 'file:read', 'file:write'];

contextBridge.exposeInMainWorld('ipc', {
  invoke: (channel, ...args) => {
    if (!ALLOWED_CHANNELS.includes(channel)) throw new Error('Blocked channel');
    return ipcRenderer.invoke(channel, ...args);
  },
});

// main.js — always validate args
ipcMain.handle('file:read', async (event, filePath) => {
  if (typeof filePath !== 'string') throw new Error('Invalid path');
  const safe = path.resolve(filePath);
  if (!safe.startsWith(ALLOWED_DIR)) throw new Error('Path traversal denied');
  return fs.promises.readFile(safe, 'utf8');
});

BrowserWindow options

▶
const win = new BrowserWindow({
  width: 1200, height: 800,
  minWidth: 600, minHeight: 400,
  frame: false,          // frameless (custom titlebar)
  titleBarStyle: 'hidden',  // macOS traffic lights only
  show: false,           // wait until ready-to-show
  backgroundColor: '#1a1a1a',
  webPreferences: {
    preload: path.join(__dirname, 'preload.js'),
    contextIsolation: true,
    sandbox: true,
    spellcheck: false,
  },
});

win.once('ready-to-show', () => win.show());  // avoid white flash
win.loadFile('index.html');
Always set show: false and reveal on ready-to-show to prevent a white flash on startup.

Dialogs, menus & system tray

▶
// Dialog
const { dialog, Menu, Tray, nativeImage } = require('electron');

const { filePaths } = await dialog.showOpenDialog(win, {
  title: 'Open File',
  filters: [{ name: 'JSON', extensions: ['json'] }],
  properties: ['openFile'],
});

// Application Menu
const menu = Menu.buildFromTemplate([
  { label: 'File', submenu: [
    { label: 'Open', accelerator: 'CmdOrCtrl+O', click: openFile },
    { type: 'separator' },
    { role: 'quit' },
  ]},
]);
Menu.setApplicationMenu(menu);

// System Tray
const tray = new Tray(nativeImage.createFromPath('icon.png'));
tray.setContextMenu(Menu.buildFromTemplate([
  { label: 'Show', click: () => win.show() },
  { label: 'Quit', click: () => app.quit() },
]));

Notifications & shell

▶
const { Notification, shell, clipboard } = require('electron');

// OS notification
new Notification({ title: 'Done', body: 'Export complete' }).show();

// Open URL in default browser (safer than loadURL)
await shell.openExternal('https://example.com');

// Open file in OS default app
await shell.openPath('/path/to/file.pdf');

// Clipboard
clipboard.writeText('copied!');
const text = clipboard.readText();
Use shell.openExternal only with validated URLs — passing untrusted user input can execute arbitrary programs (e.g. file:// paths).
Packaging & Distribution Build

electron-builder config

▶

electron-builder is the standard tool for packaging Electron apps. It produces platform-native installers (.dmg, .exe, .AppImage) and handles code signing.

// package.json
{
  "build": {
    "appId": "com.company.myapp",
    "productName": "MyApp",
    "directories": { "output": "dist" },
    "files": ["dist/**", "main.js", "preload.js"],
    "mac": {
      "category": "public.app-category.productivity",
      "hardenedRuntime": true,
      "entitlements": "entitlements.mac.plist"
    },
    "win": { "target": ["nsis"] },
    "linux": { "target": ["AppImage", "deb"] },
    "publish": { "provider": "github" }
  }
}

Auto-updates with electron-updater

▶
const { autoUpdater } = require('electron-updater');

autoUpdater.checkForUpdatesAndNotify();

// Or manual flow
autoUpdater.on('update-available', () => {
  win.webContents.send('update-available');
});
autoUpdater.on('update-downloaded', () => {
  win.webContents.send('update-downloaded');
});

// Triggered from renderer via IPC
ipcMain.on('install-update', () => autoUpdater.quitAndInstall());
Set "publish": { "provider": "github" } in electron-builder config and push a GitHub Release — electron-updater detects it automatically.
