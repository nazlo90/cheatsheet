C# Type System & Value vs Reference Types Interview

Value types vs Reference types

▶
AspectValue TypesReference Types
StorageStack (usually)Heap
AssignmentCopies dataCopies reference
Null by defaultNo (nullable<T> needed)Yes
Examplesint, bool, struct, enumclass, string, array, delegate
InheritanceSealed implicitlySupports inheritance
// Value type — copy semantics
int a = 5;
int b = a;
b = 10; // a is still 5

// Reference type — shared reference
var list1 = new List<int> { 1, 2 };
var list2 = list1;
list2.Add(3); // list1 also has 3
Boxing occurs when a value type is assigned to object — allocates heap memory. Avoid in hot paths.

Nullable value types & null-coalescing operators

▶
int? age = null;
age ??= 18;           // assign if null
int val = age ?? 0;   // fallback
string? name = null;
int len = name?.Length ?? 0; // null-conditional
OperatorMeaning
??Return left if not null, else right
??=Assign right to left only if left is null
?.Null-conditional member access
?[]Null-conditional index access

LINQ Interview

Deferred vs immediate execution

▶
Deferred (lazy)Immediate (eager)
Where, Select, OrderBy, GroupBy, Skip, TakeToList, ToArray, ToDictionary, Count, First, Any, Sum, Max
Query built, not run yetQuery executed right away
var query = users
    .Where(u => u.IsActive)        // deferred
    .OrderBy(u => u.Name)           // deferred
    .Select(u => new { u.Id, u.Name }); // deferred

var result = query.ToList(); // executes here
Enumerating a deferred query multiple times hits the source multiple times. Call ToList() to materialize when reuse is needed.

Common LINQ operators

▶
// Projection
users.Select(u => u.Name);
users.SelectMany(u => u.Orders); // flatten nested

// Filtering
users.Where(u => u.Age > 18);
users.OfType<AdminUser>();

// Aggregation
orders.Sum(o => o.Total);
orders.GroupBy(o => o.CustomerId)
      .Select(g => new { g.Key, Total = g.Sum(o => o.Total) });

// Joining
users.Join(orders, u => u.Id, o => o.UserId,
    (u, o) => new { u.Name, o.Total });

// Set operations
a.Union(b); a.Intersect(b); a.Except(b);

Query syntax vs method syntax

▶
// Method (fluent) syntax
var r1 = users
    .Where(u => u.IsActive)
    .Select(u => u.Name);

// Query syntax (compiles to same IL)
var r2 = from u in users
          where u.IsActive
          select u.Name;
Async / Await & Task Interview

async/await fundamentals

▶
// async method returns Task/Task<T>/ValueTask<T>
public async Task<User> GetUserAsync(int id)
{
    var data = await _db.QueryAsync(id); // releases thread
    return new User(data);
}

// Run multiple tasks in parallel
var (user, orders) = await (Task, Task).WhenAll(
    GetUserAsync(id),
    GetOrdersAsync(id));

// ValueTask — avoid allocation for sync-fast paths
public async ValueTask<int> ReadCachedAsync()
{
    if (_cache.TryGetValue(out var v)) return v;
    return await _db.ReadAsync();
}
Never use async void except for event handlers — exceptions can't be caught and will crash the process.

Exception Handling & IDisposable

try / catch / finally

▶
try
{
    await _db.SaveAsync(entity);
}
catch (DbUpdateConcurrencyException ex)
{
    _logger.LogWarning(ex, "Concurrency conflict for {Id}", entity.Id);
    throw; // re-throw preserves stack trace
}
catch (Exception ex) when (ex.Message.Contains("timeout")) // exception filter
{
    throw new ServiceUnavailableException("DB timeout", ex);
}
finally
{
    _conn.Close(); // always runs
}

IDisposable & using

▶
// using statement — calls Dispose() on scope exit
using var conn = new SqlConnection(connStr);
await conn.OpenAsync();

// Implementing IDisposable correctly
public class FileProcessor : IDisposable
{
    private Stream? _stream;
    private bool _disposed;

    public void Dispose()
    {
        if (_disposed) return;
        _stream?.Dispose();
        _disposed = true;
    }
}
Prefer await using for IAsyncDisposable (e.g., EF Core DbContext).

Pattern Matching (C# 8–12) Interview

switch expressions & patterns

▶
// Type pattern + property pattern
string Describe(object obj) => obj switch
{
    Circle { Radius: > 10 } c => $"Large circle r={c.Radius}",
    Circle c                  => $"Small circle r={c.Radius}",
    Rectangle { Width: var w, Height: var h } => $"Rect {w}x{h}",
    null                       => "null",
    _                          => "unknown"
};

// Relational & logical patterns
string Category(int score) => score switch
{
    >= 90           => "A",
    >= 70 and < 90 => "B",
    >= 50           => "C",
    _               => "F"
};
Memory Management & Span<T>

Span<T>, Memory<T>, ArrayPool

▶
// Span — stack-only, zero-copy slice over contiguous memory
void Process(Span<byte> data)
{
    var header = data[..4];
    var body   = data[4..];
}

// Memory<T> — heap-safe, usable across async boundaries
async Task ReadAsync(Memory<byte> buffer)
{
    await _stream.ReadAsync(buffer);
}

// ArrayPool — reuse buffers, avoid GC pressure
var pool = ArrayPool<byte>.Shared;
byte[] buf = pool.Rent(4096);
try { /* use buf */ }
finally { pool.Return(buf); }
