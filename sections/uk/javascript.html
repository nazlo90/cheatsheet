Hoisting & var / let / const Розминка

Правила Hoisting та TDZ

▶

Hoisting — це поведінка JS, яка переміщує оголошення на початок їхньої області видимості перед виконанням — тому змінні та функції можна використовувати до відповідного рядка коду.

JS-рушій виконує двопрохідне сканування: спочатку збирає оголошення, потім виконує код. Підіймаються лише оголошення — не ініціалізації.

Event Loop Основне

Call Stack → Мікрозадачі → Макрозадачі

▶

Event loop — це механізм, який дозволяє JS (однопотоковому) обробляти асинхронні операції — він постійно перевіряє, чи порожній call stack, а потім обробляє задачі з черги.

  1. Call Stack — тут виконується синхронний код
  2. Черга мікрозадач — Promise.then/catch/finally, queueMicrotask(), MutationObserver
  3. Черга макрозадач — setTimeout, setInterval, події UI, I/O (по одній за тік циклу)
Після кожної макрозадачі вся черга мікрозадач повністю спустошується перед наступною макрозадачею.
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Результат: 1 → 4 → 3 → 2

▶

Замикання — визначення та випадки використання

▶

Замикання зберігає доступ до змінних зовнішньої області видимості навіть після того, як зовнішня функція повернула результат.

function makeCounter(init = 0) {
  let count = init;
  return { inc: () => ++count, dec: () => --count, val: () => count };
}
const c = makeCounter(10);
c.inc(); // 11
Випадки використання: шаблон модуля, мемоізація, часткове застосування, обробники подій, інкапсуляція даних, фабричні функції.
Функції вищого порядку

HOF, каррування, мемоізація, композиція

▶

HOF або приймає функцію як аргумент, або повертає функцію.

// Каррування: перетворює функцію з кількома аргументами на послідовність функцій з одним аргументом.
const add = a => b => a + b;
const add5 = add(5);
add5(3); // 8

// Загальне каррування: адаптує будь-яку функцію — збирає аргументи до досягнення арності, потім викликає.
const curry = fn => function curried(...args) {
  return args.length >= fn.length ? fn(...args) : (...more) => curried(...args, ...more);
};

// Мемоізація: кешує значення за серіалізованими аргументами для уникнення повторних обчислень.
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const r = fn.apply(this, args);
    cache.set(key, r); return r;
  };
}

// Композиція та pipe: compose = справа-наліво (f(g(x))), pipe = зліва-направо (спочатку g, потім f).
const compose = (...fns) => x => fns.reduceRight((v, f) => f(v), x);
const pipe    = (...fns) => x => fns.reduce((v, f) => f(v), x);

Класи (повний огляд)

Синтаксис класів — повний довідник

▶

Клас — це синтаксичний цукор над прототипним успадкуванням — шаблон для створення об'єктів зі спільними методами, приватними полями та логікою конструктора. Під капотом це все одно ланцюжки прототипів.

class Animal {
  // Поле класу (ES2022)
  #name;                          // приватне поле
  species = 'unknown';           // публічне поле
  static count = 0;              // статичне поле

  constructor(name) {
    this.#name = name;
    Animal.count++;
  }

  // Гетер / Сетер
  get name() { return this.#name; }
  set name(v) { this.#name = v; }

  speak() { return `${this.#name} makes noise`; }
  static getCount() { return Animal.count; }
}

class Dog extends Animal {
  #tricks = [];

  constructor(name) {
    super(name);           // спочатку потрібно викликати super
  }

  speak() {
    return super.speak() + ' — Woof!';
  }

  learn(trick) { this.#tricks.push(trick); }
}

// instanceof працює через ланцюжок прототипів
new Dog('Rex') instanceof Animal; // true

Promise та async/await

Стани Promise, ланцюжок, обробка помилок

▶

Promise — це об'єкт, що представляє кінцеве виконання або невдачу асинхронної операції. Стани: pending → fulfilled | rejected (незмінні після вирішення).

// Створення: огортає асинхронну операцію — викликає resolve при успіху, reject при помилці.
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('done'), 1000);
});

// Ланцюжок — кожен .then повертає новий Promise
p.then(v => v.toUpperCase())
 .then(v => console.log(v))
 .catch(err => console.error(err))
 .finally(() => console.log('done'));

// Статичні методи: комбінатори для координації кількох Promise — відрізняються обробкою відхилень.
Promise.all([p1, p2]);          // fail-fast: відхиляється, якщо будь-який відхилено
Promise.allSettled([p1, p2]);   // чекає всіх, повертає {status, value/reason}[]
Promise.any([p1, p2]);          // перший виконаний
Promise.race([p1, p2]);         // перший вирішений (виконаний АБО відхилений)
Promise.resolve(val);
Promise.reject(err);

// Основи: async/await робить асинхронний код схожим на синхронний — завжди загортайте в try/catch.
async function fetchUser(id) {
  try {

Генератори та ітератори

Функції-генератори

▶

Функція-генератор повертає об'єкт Generator. Виконання зупиняється на кожному yield і відновлюється при виклику .next().

function* counter(start = 0) {
  while (true) {
    const reset = yield start;  // може отримувати значення через .next(val)
    start = reset ?? start + 1;
  }
}
const gen = counter(1);
gen.next();         // { value: 1, done: false }
gen.next();         // { value: 2, done: false }
gen.next(10);       // { value: 10, done: false } — скидання

// Скінченний генератор
function* range(start, end, step = 1) {
  for (let i = start; i < end; i += step) yield i;
}
[...range(0, 10, 2)]; // [0, 2, 4, 6, 8]
