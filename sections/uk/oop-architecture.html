Огляд парадигм програмування

▶

Парадигма — це фундаментальний стиль побудови програм, що визначає підхід до стану, потоку управління та композиції. Більшість сучасних мов є мультипарадигменними.

ПроцедурнаІмперативна + багаторазові процедури/функціїГрупування кроків у іменовані підпрограмиC, Pascal, ранні JS-скрипти

Функціональна (ФП)Перетворення даних через чисті функції, уникнення побічних ефектівПайплайни перетворень; дані незмінніHaskell, Elm, RxJS, Redux, сучасний JS/TS

Композиція функцій, каррінг та часткове застосування

▶

Композиція з'єднує функції так, що вихід однієї стає входом наступної — будуючи складну поведінку з маленьких, повторно використовуваних частин.

Каррінг перетворює функцію з кількома аргументами на послідовність функцій з одним аргументом. Часткове застосування фіксує деякі аргументи наперед і повертає функцію для решти.

// Композиція — f(g(x))
const compose = <T>(...fns: Array<(x: T) => T>) =>
  (x: T) => fns.reduceRight((v, f) => f(v), x);

const pipe = <T>(...fns: Array<(x: T) => T>) =>
  (x: T) => fns.reduce((v, f) => f(v), x);

const double  = (n: number) => n * 2;
const addOne  = (n: number) => n + 1;
const transform = pipe(double, addOne);  // (5) → 10 → 11

// Каррінг
const multiply = (a: number) => (b: number) => a * b;
const triple   = multiply(3);  // часткове застосування — b ще вільний
triple(4);  // 12

// pipe у RxJS — це буквально композиція функцій
obs$.pipe(
  filter(x => x > 0),
  map(x => x * 2),
  take(5)
);
RxJS pipe(), Angular pipe у шаблонах, та ланцюжки Array.prototype (.filter().map().reduce()) — усе це функціональна композиція у дії.

Рекурсія та оптимізація хвостових викликів

▶

Рекурсія замінює явні цикли у ФП — функція викликає саму себе з меншою підзадачею. Чисто функціональні мови покладаються на неї.

Оптимізація хвостових викликів (TCO): якщо рекурсивний виклик є останньою операцією, рушій може повторно використати поточний стек-фрейм замість додавання нового — запобігаючи переповненню стека. ES6 вимагає TCO у строгому режимі (V8 відмовився від цього; для глибокої рекурсії у JS надавайте перевагу ітерації або трамплінам).

// Не хвостово-рекурсивна — стек зростає O(n)
function factorial(n: number): number {
  return n <= 1 ? 1 : n * factorial(n - 1);
}

// Хвостово-рекурсивна (патерн акумулятора)
function factTCO(n: number, acc = 1): number {
  return n <= 1 ? acc : factTCO(n - 1, n * acc);
}

// Безпечна глибока рекурсія у JS — патерн трамплін
function trampoline<T>(fn: (...args: any[]) => T | (() => T)) {
  return function(...args: any[]) {
    let result: any = fn(...args);
    while (typeof result === 'function') result = result();
    return result;
  };
}

▶

Nx — це розумна система збірки та інструмент монорепозиторію, що допомагає керувати кількома аплікаціями та спільними бібліотеками в одному репозиторії.

ГенераториСтандартизоване створення аплікацій, бібліотек, компонентів
