Архітектура NestJS Основне

Модулі, Контролери, Провайдери

▶

NestJS організовує код у Модулі. Кожен модуль інкапсулює окрему функціональну частину. Фреймворк зв'язує залежності через свій IoC-контейнер.

Будівельний блокДекораторРоль
Module@Module()Об'єднує провайдери та контролери; визначає, що експортується
Controller@Controller()Обробляє вхідні запити, зіставляє маршрути, повертає відповіді
Provider / Service@Injectable()Бізнес-логіка; впроваджується через DI конструктора
Guard@UseGuards()Перевірка авторизації перед виконанням обробника
Pipe@UsePipes()Валідація / перетворення даних запиту
Interceptor@UseInterceptors()AOP: логування, кешування, формування відповіді
Filter@UseFilters()Обробка виключень — перетворює помилки на HTTP-відповіді
// users.module.ts
import { Module } from '@nestjs/common';
import { UsersController } from './users.controller';
import { UsersService } from './users.service';

@Module({
  controllers: [UsersController],
  providers:   [UsersService],
  exports:     [UsersService],  // робить його доступним для модулів, що імпортують цей
})
export class UsersModule {}
Тримайте модулі в межах окремих функціональних областей. AppModule — кореневий; функціональні модулі імпортуються до нього (або ліниво — для мікросервісів).

Контролер і декоратори маршрутів

▶
import { Controller, Get, Post, Body, Param, Query,
         HttpCode, HttpStatus, Delete, Patch } from '@nestjs/common';
import { UsersService } from './users.service';
import { CreateUserDto } from './dto/create-user.dto';

@Controller('users')   // префікс → /users
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll(@Query('role') role?: string) {
    return this.usersService.findAll(role);
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.usersService.findOne(+id);
  }

  @Post()
  @HttpCode(HttpStatus.CREATED)
  create(@Body() dto: CreateUserDto) {
    return this.usersService.create(dto);
  }

  @Patch(':id')
  update(@Param('id') id: string, @Body() dto: Partial<CreateUserDto>) {
    return this.usersService.update(+id, dto);
  }

  @Delete(':id')
  @HttpCode(HttpStatus.NO_CONTENT)
  remove(@Param('id') id: string) {
    return this.usersService.remove(+id);
  }
}
Контролери NestJS мають бути тонкими — розміщуйте бізнес-логіку в сервісах. Контролери лише зв'язують HTTP із викликами сервісів.
Впровадження залежностей та Провайдери DI

Типи провайдерів та власні токени

▶

Провайдер — це будь-що, зареєстроване в масиві providers модуля. IoC-контейнер вирішує їх за токеном.

// Стандартний провайдер класу (скорочена форма)
providers: [UsersService]

// Еквівалентна повна форма
providers: [{ provide: UsersService, useClass: UsersService }]

// Value провайдер — об'єкт конфігурації, константи
providers: [{ provide: 'CONFIG', useValue: { db: 'postgres://...' } }]

// Factory провайдер — асинхронна ініціалізація (з'єднання з БД тощо)
providers: [{
  provide: 'DB_CONN',
  useFactory: async (cfg: ConfigService) => {
    return await createConnection(cfg.get('DATABASE_URL'));
  },
  inject: [ConfigService],
}]

// Впровадження токена-значення або некласового токена
@Injectable()
export class AppService {
  constructor(@Inject('CONFIG') private cfg: any) {}
}
Використовуйте об'єкти InjectionToken замість рядків для типобезпечності: const CONFIG = new InjectionToken<Config>('CONFIG').

Скоупи: Singleton vs Request vs Transient

▶
СкоупЧас життяКоли використовувати
DEFAULT (singleton)Час життя застосунку, спільний екземплярStateless-сервіси, репозиторії
REQUESTОдин екземпляр на HTTP-запитКонтекст на рівні запиту (користувач, тенант)
TRANSIENTНовий екземпляр при кожному впровадженніStateful-утиліти, що не повинні ділити стан
import { Injectable, Scope } from '@nestjs/common';

@Injectable({ scope: Scope.REQUEST })
export class RequestScopedService {}
REQUEST-скоупні провайдери «спливають» вгору — будь-який провайдер, що впроваджує REQUEST-скоупний, сам стає REQUEST-скоупним. Це впливає на продуктивність; надавайте перевагу singleton + AsyncLocalStorage для контексту на рівні запиту.
Валідація та Pipes Практичне

DTO з class-validator + ValidationPipe

▶
// create-user.dto.ts
import { IsEmail, IsString, MinLength, IsOptional, IsEnum } from 'class-validator';
import { Transform } from 'class-transformer';

export enum Role { ADMIN = 'admin', USER = 'user' }

export class CreateUserDto {
  @IsString()
  @MinLength(2)
  name: string;

  @IsEmail()
  @Transform(({ value }) => value.toLowerCase())
  email: string;

  @IsOptional()
  @IsEnum(Role)
  role?: Role;
}

// main.ts — увімкнути глобально
app.useGlobalPipes(new ValidationPipe({
  whitelist: true,       // видаляти невідомі властивості
  forbidNonWhitelisted: true,
  transform: true,       // автоматично приводити payload до класів DTO
}));
Завжди встановлюйте whitelist: true, щоб запобігти передачі клієнтами зайвих полів, які можуть випадково перезаписати властивості.

Власний Pipe та ParseIntPipe

▶
import { PipeTransform, Injectable, ArgumentMetadata, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParsePositiveIntPipe implements PipeTransform {
  transform(value: string, metadata: ArgumentMetadata) {
    const n = parseInt(value, 10);
    if (isNaN(n) || n <= 0) throw new BadRequestException('Must be a positive integer');
    return n;
  }
}

// Використання
@Get(':id')
findOne(@Param('id', ParsePositiveIntPipe) id: number) { ... }

Guards та Автентифікація Безпека

JWT Auth Guard з Passport

▶
// jwt.strategy.ts
import { PassportStrategy } from '@nestjs/passport';
import { Strategy, ExtractJwt } from 'passport-jwt';

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(config: ConfigService) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      secretOrKey:    config.get('JWT_SECRET'),
    });
  }
  validate(payload: { sub: number; email: string }) {
    return { id: payload.sub, email: payload.email }; // прикріплюється до req.user
  }
}

// jwt-auth.guard.ts
import { AuthGuard } from '@nestjs/passport';
export class JwtAuthGuard extends AuthGuard('jwt') {}

// Використання в контролері
@UseGuards(JwtAuthGuard)
@Get('profile')
getProfile(@Request() req) { return req.user; }
Зареєструйте JwtStrategy як провайдер і імпортуйте PassportModule + JwtModule.registerAsync() у AuthModule.

Рольовий доступ через власний Guard та Reflector

▶
// roles.decorator.ts
export const Roles = (...roles: string[]) => SetMetadata('roles', roles);

// roles.guard.ts
@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(ctx: ExecutionContext): boolean {
    const required = this.reflector.getAllAndOverride<string[]>('roles', [
      ctx.getHandler(), ctx.getClass(),
    ]);
    if (!required) return true;
    const { user } = ctx.switchToHttp().getRequest();
    return required.some(r => user.roles?.includes(r));
  }
}

// Використання
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('admin')
@Delete(':id')
remove(@Param('id') id: string) { ... }
Interceptors, Filters та Обробка виключень AOP

Interceptor для трансформації відповіді

▶
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { map } from 'rxjs/operators';

@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(ctx: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(map(data => ({ data, success: true })));
  }
}

// Logging interceptor — вимірювання часу виконання обробника
@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(ctx: ExecutionContext, next: CallHandler): Observable<any> {
    const now = Date.now();
    return next.handle().pipe(tap(() => console.log(`Handler took ${Date.now() - now}ms`)));
  }
}

Фільтри виключень

▶
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx  = host.switchToHttp();
    const res  = ctx.getResponse();
    const req  = ctx.getRequest();
    const status = exception.getStatus();

    res.status(status).json({
      statusCode: status,
      timestamp:  new Date().toISOString(),
      path:       req.url,
      message:    exception.message,
    });
  }
}

// Перехоплювати все — корисно для необроблених помилок
@Catch()
export class AllExceptionsFilter implements ExceptionFilter { ... }

// Зареєструвати глобально у main.ts
app.useGlobalFilters(new HttpExceptionFilter());
Nest постачається з вбудованими HTTP-виключеннями: NotFoundException, BadRequestException, UnauthorizedException, ForbiddenException, ConflictException тощо.
Робота з базою даних через TypeORM Дані

Entity, Repository та транзакції

▶
// user.entity.ts
import { Entity, PrimaryGeneratedColumn, Column, CreateDateColumn } from 'typeorm';

@Entity('users')
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  email: string;

  @Column()
  name: string;

  @CreateDateColumn()
  createdAt: Date;
}

// users.service.ts
@Injectable()
export class UsersService {
  constructor(
    @InjectRepository(User)
    private repo: Repository<User>,
    private dataSource: DataSource,
  ) {}

  findAll() { return this.repo.find(); }

  // Транзакція через DataSource
  async transferCredits(fromId: number, toId: number, amount: number) {
    return this.dataSource.transaction(async manager => {
      await manager.decrement(User, { id: fromId }, 'credits', amount);
      await manager.increment(User, { id: toId },   'credits', amount);
    });
  }
}
Ніколи не використовуйте synchronize: true у production — це виконує автоміграцію при запуску та може призвести до втрати даних. Натомість використовуйте міграції TypeORM CLI.

const users = await this.repo

// Зв'язки на рівні entity

Жадібне завантаження всіх зв'язків за замовчуванням спричиняє N+1 запити. Використовуйте опцію relations у find() або явний leftJoinAndSelect у QueryBuilder.
Конфігурація та Середовище Ops

ConfigModule та типізована конфігурація

▶
// app.module.ts
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,     // не потрібно імпортувати у кожен модуль
      envFilePath: '.env',
      validationSchema: Joi.object({
        PORT:         Joi.number().default(3000),
        DATABASE_URL: Joi.string().required(),
        JWT_SECRET:   Joi.string().min(32).required(),
      }),
    }),
  ],
})
export class AppModule {}

// Типізований простір імен конфігурації
export const dbConfig = registerAs('db', () => ({
  url:  process.env.DATABASE_URL,
  pool: parseInt(process.env.DB_POOL ?? '5', 10),
}));

// Впровадження в сервіс
@Inject(dbConfig.KEY) private db: ConfigType<typeof dbConfig>
Перевіряйте всі змінні середовища при запуску за допомогою validationSchema. Краще одразу падати з помилкою, ніж отримати збій у runtime через відсутню змінну.
Тестування NestJS Якість

Модульне тестування з Test.createTestingModule

▶
import { Test, TestingModule } from '@nestjs/testing';
import { UsersService } from './users.service';
import { getRepositoryToken } from '@nestjs/typeorm';

describe('UsersService', () => {
  let service: UsersService;
  const mockRepo = { find: jest.fn(), findOne: jest.fn(), save: jest.fn() };

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [
        UsersService,
        { provide: getRepositoryToken(User), useValue: mockRepo },
      ],
    }).compile();

    service = module.get<UsersService>(UsersService);
  });

  it('should return all users', async () => {
    mockRepo.find.mockResolvedValue([{ id: 1, name: 'Alice' }]);
    const result = await service.findAll();
    expect(result).toHaveLength(1);
  });
});
Для інтеграційних та e2e-тестів використовуйте supertest разом із повноцінним NestApplication. Тримайте модульні тести зосередженими на сервісах; нехай e2e-тести покривають HTTP-шар.
