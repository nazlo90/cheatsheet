▶

Angular — це фреймворк на основі TypeScript для побудови SPA з компонентною архітектурою, вбудованим DI, маршрутизацією, реактивними формами та потужним CLI.

Angular Directives Angular

Що таке Directive та її типи

▶

Directive — це клас, декорований @Directive, що додає поведінку або структуру до DOM-елементів. Component — це особливий вид Directive з шаблоном.

ТипПризначенняПриклади
ComponentDirective + шаблон + стилі@Component — кожен Component
StructuralЗмінює структуру DOM (додає/видаляє елементи)*ngIf, *ngFor, *ngSwitch, @if, @for
AttributeЗмінює зовнішній вигляд або поведінку елемента-хостаNgClass, NgStyle, RouterLink, власні
// Attribute directive — підсвічування при наведенні
@Directive({
  selector: '[appHighlight]',
  standalone: true
})
export class HighlightDirective {
  @Input('appHighlight') color = 'yellow';

  private el = inject(ElementRef);

  @HostListener('mouseenter') onEnter() {
    this.el.nativeElement.style.backgroundColor = this.color;
  }
  @HostListener('mouseleave') onLeave() {
    this.el.nativeElement.style.backgroundColor = '';
  }
}

<!-- <p [appHighlight]="'lightblue'">Hover me</p> -->

Структурні Directive — мікросинтаксис та власні

▶

Структурні Directive використовують синтаксичний цукор * — *ngIf="x" розгортається до [ngIf]="x" на прихованому <ng-template>. Власні структурні Directive впроваджують TemplateRef та ViewContainerRef.

// Власна структурна Directive — *appUnless (протилежна до *ngIf)
@Directive({ selector: '[appUnless]', standalone: true })
export class UnlessDirective {
  private vcr = inject(ViewContainerRef);
  private tmpl = inject(TemplateRef);
  private hasView = false;

  @Input() set appUnless(condition: boolean) {
    if (!condition && !this.hasView) {
      this.vcr.createEmbeddedView(this.tmpl);
      this.hasView = true;
    } else if (condition && this.hasView) {
      this.vcr.clear();
      this.hasView = false;
    }
  }
}

<!-- <div *appUnless="isLoggedIn">Please log in</div> -->

// Сучасні вбудовані еквіваленти (синтаксис шаблонів Angular 17+)
// @if (show) { <p>visible</p> } @else { <p>hidden</p> }
// @for (item of items; track item.id) { <li>{{ item.name }}</li> }
// @switch (status) { @case ('active') { <span>Active</span> } }
Надавайте перевагу сучасному синтаксису @if/@for/@switch (Angular 17+) перед *ngIf/*ngFor — він компілюється ефективніше (без зайвої DOM-обгортки) та має чистіший синтаксис.

HostBinding, HostListener та exportAs

▶
// @HostBinding — прив'язує властивість Directive до атрибута/класу/стилю елемента-хоста
@Directive({ selector: '[appActive]', standalone: true })
export class ActiveDirective {
  @Input() appActive = false;

  @HostBinding('class.active') get isActive() { return this.appActive; }
  @HostBinding('attr.aria-selected') get ariaSelected() { return this.appActive; }
  @HostBinding('style.fontWeight') get fw() { return this.appActive ? 'bold' : 'normal'; }
}

// exportAs — надає доступ до екземпляра Directive через змінну шаблону
@Directive({ selector: '[appTooltip]', exportAs: 'tooltip', standalone: true })
export class TooltipDirective {
  visible = false;
  show() { this.visible = true; }
  hide() { this.visible = false; }
}

Pipe — це функція трансформації для шаблону: приймає вхідне значення та необов'язкові аргументи й повертає перетворене значення. Використовується з оператором | у шаблонах.

// Власний чистий Pipe — форматування розміру файлу
@Pipe({ name: 'fileSize', standalone: true })
export class FileSizePipe implements PipeTransform {
  transform(bytes: number, decimals = 2): string {
    if (!bytes) return '0 B';
    const k = 1024;
    const units = ['B', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return `${(bytes / k ** i).toFixed(decimals)} ${units[i]}`;
  }
}
<!-- {{ 1048576 | fileSize }} → '1.00 MB' -->

// Власний нечистий Pipe — фільтрація змінного масиву
@Pipe({ name: 'filterBy', standalone: true, pure: false })
export class FilterByPipe implements PipeTransform {
  transform<T>(items: T[], key: keyof T, value: unknown): T[] {
    return items.filter(item => item[key] === value);
  }
}
<!-- {{ users | filterBy:'role':'admin' }} -->

// Pipe з DI — впровадження сервісів у Pipe
@Pipe({ name: 'translate', standalone: true })
export class TranslatePipe implements PipeTransform {
  private i18n = inject(I18nService);
  transform(key: string): string { return this.i18n.get(key); }
}
Уникайте нечистих Pipe-ів на великих списках — вони запускаються при кожному циклі виявлення змін. Надавайте перевагу чистим Pipe-ам із незмінними даними або використовуйте сигнал computed() для реактивної фільтрації.
Dependency Injection Angular

Система DI — ієрархічні інжектори

▶

Angular має ієрархічну систему DI. Кожен Component має власний інжектор; якщо залежність не знайдена, відбувається підйом до батьківського інжектора, потім до кореневого.

Ієрархія інжекторів: Platform → Root (AppModule) → Feature Module → Component → Element

// Singleton у кореневому інжекторі
@Injectable({ providedIn: 'root' })   // підтримує tree-shaking
class AuthService {}

// Область видимості Component (новий екземпляр для кожного Component)
@Component({ providers: [MyService] })
class MyComponent {}

// InjectionToken — для значень, що не є класами
const API_URL = new InjectionToken<string>('API_URL');
// providers: [{ provide: API_URL, useValue: 'https://...' }]
inject(API_URL); // функціональне впровадження (Angular 14+)

// @Optional — null, якщо не знайдено
constructor(@Optional() private logger?: LoggerService) {}

// @Self — шукає лише у власному інжекторі
constructor(@Self() private svc: MyService) {}

// @SkipSelf — пропускає власний, шукає у батьківському
constructor(@SkipSelf() private parent: MyService) {}

// @Host — зупиняється на межі елемента-хоста
constructor(@Host() private svc: MyService) {}

// Factory provider — використовуйте, коли створення потребує логіки або асинхронної ініціалізації
{ provide: Logger, useFactory: (http: HttpClient) => new RemoteLogger(http), deps: [HttpClient] }

// Multi-provider (interceptors, validators)
{ provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }

Factory provider — патерни та асинхронна ініціалізація

▶

useFactory надає повний контроль над створенням екземпляра — приймає впроваджені залежності, може містити умовну логіку та є єдиним типом провайдера, що підтримує асинхронну ініціалізацію через APP_INITIALIZER.

// 1. Умовна фабрика — обирає реалізацію залежно від середовища
const loggerProvider = {
  provide: Logger,
  useFactory: (env: Environment) =>
    env.production ? new RemoteLogger() : new ConsoleLogger(),
  deps: [Environment]
};

// 2. Асинхронна фабрика через APP_INITIALIZER — блокує завантаження застосунку до завершення
export function initializeApp(configService: ConfigService) {
  return () => configService.load();  // повертає Promise або Observable
}

providers: [
  ConfigService,
  {
    provide: APP_INITIALIZER,
    useFactory: initializeApp,
    deps: [ConfigService],
    multi: true
  }
]

// 3. Фабрика InjectionToken (функціональний стиль Angular 14+)
const BASE_URL = new InjectionToken<string>('BASE_URL', {
  providedIn: 'root',
  factory: () => inject(DOCUMENT).location.origin
});
Усі залежності у масиві deps мають бути зареєстровані в тому самому або батьківському інжекторі. Для необов'язкових залежностей використовуйте new Optional() у масиві deps: deps: [HttpClient, new Optional(), SomeOptionalDep]

Multi-providers — збирання кількох реалізацій

▶

multi: true вказує Angular накопичувати всі провайдери для токена в масив замість заміни попереднього. Впроваджене значення має тип T[].

// Вбудовані multi-provider токени
// HTTP_INTERCEPTORS   → HttpInterceptor[]
// APP_INITIALIZER     → (() => Promise|Observable)[]
// NG_VALIDATORS       → Validator[]

// Власний токен плагіна
interface DashboardWidget { render(): string; }
const DASHBOARD_WIDGETS = new InjectionToken<DashboardWidget[]>('DASHBOARD_WIDGETS');

providers: [
  { provide: DASHBOARD_WIDGETS, useClass: SalesWidget,   multi: true },
  { provide: DASHBOARD_WIDGETS, useClass: UsersWidget,   multi: true },
  { provide: DASHBOARD_WIDGETS, useClass: RevenueWidget, multi: true },
]

// Впровадження як масив
class DashboardComponent {
  widgets = inject(DASHBOARD_WIDGETS);  // DashboardWidget[]
}

// Власний multi-provider NG_VALIDATORS
@Directive({ selector: '[appForbiddenName]', providers: [
  { provide: NG_VALIDATORS, useExisting: ForbiddenNameDirective, multi: true }
]})
class ForbiddenNameDirective implements Validator {
  validate(control: AbstractControl): ValidationErrors | null { /* ... */ }
}
Використовуйте useExisting (а не useClass) при реєстрації Directive як валідатора — useClass створює другий екземпляр; useExisting повторно використовує той, що вже створений Angular.

Виявлення змін (Change Detection) Angular

Default проти OnPush + Zone.js

▶

Zone.js патчить усі асинхронні API та сповіщає Angular після виконання асинхронних завдань. Default: перевіряє все дерево. OnPush: пропускає піддерево, якщо не виконано одну з умов:

  • Змінюється посилання на @Input() (не мутація)
  • Подія відбувається всередині Component
  • Observable видає значення через async pipe
  • Викликано markForCheck() / detectChanges()
@Component({ changeDetection: ChangeDetectionStrategy.OnPush })
class CardComponent {
  @Input() data!: Data;   // необхідно передати НОВЕ посилання

  constructor(private cd: ChangeDetectorRef) {}

  update() {
    this.cd.markForCheck();    // запланувати перевірку в наступному циклі
    // АБО
    this.cd.detectChanges();   // негайна синхронна перевірка
  }
  pauseCD()  { this.cd.detach(); }
  resumeCD() { this.cd.reattach(); }

Порівняння коду — одна функція, два підходи

// ─── Підхід RxJS / Zone.js ────────────────────────────────
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    <div *ngFor="let u of users$ | async">{{ u.name }}</div>
    <span>{{ count$ | async }}</span>
  `
})
class UserListComponent {
  users$ = this.http.get<User[]>('/api/users').pipe(shareReplay(1));
  count$ = this.users$.pipe(map(u => u.length));
  // async pipe: підписується + викликає markForCheck() при кожному emitted значенні
  // Zone.js: запускає виявлення змін після HTTP, setTimeout, подій...
  // з OnPush: перевіряє лише ЦЕ піддерево, а не все дерево
}

// ─── Підхід Signals ───────────────────────────────────────
@Component({
  template: `
    <div *ngFor="let u of users()">{{ u.name }}</div>
    <span>{{ count() }}</span>
  `
})
class UserListComponent {
  users = signal<User[]>([]);
  count = computed(() => this.users().length);  // автоматично відстежує users

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<User[]>('/api/users').subscribe(u => this.users.set(u));
    // АБО: users = toSignal(this.http.get(...), { initialValue: [] });
  }
  // Angular відстежує, які вирази шаблону читають users()
  // При users.set() → лише ці прив'язування перераховуються, нічого більше
  // Не потрібні Zone.js, markForCheck()
}

RxJS interop — toSignal / toObservable

▶
import { toSignal, toObservable } from '@angular/core/rxjs-interop';

// Observable → Signal (підписується автоматично, завершується при знищенні)
class SearchComponent {
  query = signal('');

  // Signal → Observable → застосування операторів RxJS → назад у Signal
  results = toSignal(
    toObservable(this.query).pipe(
      debounceTime(300),
      distinctUntilChanged(),
      switchMap(q => this.http.get(`/search?q=${q}`))
    ),
    { initialValue: [] }
  );
  // шаблон: <li *ngFor="let r of results()">{{ r.name }}</li>
  // Найкраще з обох світів: оператори RxJS + Signals у шаблоні
}

// Правило великого пальця:
// - Використовуйте RxJS для складних асинхронних потоків (debounce, retry, об'єднання потоків)
// - Використовуйте Signals для стану UI, похідних значень, прив'язувань шаблону
// - З'єднуйте через toSignal() на межі
Шлях міграції: залиште RxJS для бізнес-логіки, загорніть у toSignal() на межі Component. Поступово замінюйте простий стан BehaviorSubject на Signals.

signal / computed / effect

▶

Signal — це реактивний примітив, що зберігає значення та автоматично сповіщає споживачів при змінах — забезпечує точне виявлення змін без Zone.js.

import { signal, computed, effect } from '@angular/core';

// Записуваний Signal
const count = signal(0);
count.set(5);
count.update(v => v + 1);

// Computed — автоматично відстежує залежності
const doubled = computed(() => count() * 2);

// Effect — виконує побічні ефекти при зміні залежностей
effect(() => console.log('count changed:', count()));

// У шаблоні
// <p>{{ count() }}</p>  <!-- виклик як функція -->

// Signal inputs (Angular 17.1+)
import { input, output, model } from '@angular/core';
class MyComponent {
  name = input<string>('');        // замінює @Input()
  nameChange = output<string>();  // замінює @Output()
  value = model<string>('');      // двостороннє прив'язування
}

    name:  ['', [Validators.required, Validators.minLength(2)]],

      debounceTime(300),
      distinctUntilChanged(),
      switchMap(email => this.api.checkEmail(email))
    ).subscribe(taken => { ... });

    // Програмні оновлення
    this.form.patchValue({ name: 'Alice' });  // часткове

// Синхронний власний валідатор
function noSpacesValidator(control: AbstractControl): ValidationErrors | null {
  return control.value?.includes(' ')
    ? { noSpaces: true }
    : null;
}

// Асинхронний валідатор (перевірка унікальності email)
function emailTakenValidator(api: ApiService): AsyncValidatorFn {
  return (control) => of(control.value).pipe(
    debounceTime(300),
    switchMap(v => api.emailExists(v)),
    map(exists => exists ? { emailTaken: true } : null)
  );
}

// Міжпольовий валідатор (на рівні групи)
function passwordMatch(group: AbstractControl): ValidationErrors | null {
  const pwd = group.get('password')?.value;
  const confirm = group.get('confirmPassword')?.value;
  return pwd === confirm ? null : { mismatch: true };
}

// Використання
fb.group({ password: [''], confirmPassword: [''] }, { validators: passwordMatch })

▶

@defer ліниво завантажує Component/блок та його залежності — це управляється компілятором, а не рантаймом. Чанк завантажується лише при спрацюванні тригера.

Incremental DOM — як Ivy оновлює DOM

▶

Ivy використовує підхід incremental DOM: інструкції генеруються на рівні Component та обходять DOM на місці, а не створюють дерево віртуального DOM. Це означає:

  • Пам'ять — немає повного снапшоту віртуального DOM у пам'яті; виділяється лише те, що змінюється
  • Tree-shaking — невикористані інструкції рендерингу (наприклад, ɵɵlistener, ɵɵtext) видаляються з бандла
  • Розмір бандла — малі застосунки отримують найбільший виграш; рантайм масштабується з використовуваними функціями, а не розміром фреймворку
  • SSR — той самий код Component рендериться на сервері через platform-server; provideClientHydration() повторно використовує серверний DOM без повного повторного рендерингу
// Скомпільований Ivy Component генерує послідовності інструкцій на кшталт:
// ɵɵelementStart(0, 'div')  ɵɵtext(1)  ɵɵelementEnd()
// ɵɵadvance(1); ɵɵtextInterpolate(ctx.title)
// Вони виконуються В ІСНУЮЧОМУ DOM — алгоритм порівняння не потрібен

// Non-destructive hydration (Angular 17+)
bootstrapApplication(AppComponent, {
  providers: [provideClientHydration()]  // повторно використовує SSR DOM, без мерехтіння
});
Локальність Ivy дозволяє бібліотекам Angular постачатися попередньо скомпільованими. strictTemplates: true у tsconfig виявляє помилки типів шаблону під час збирання — використовуйте це.

Глобальний ErrorHandler + логування через HTTP interceptor

▶
// Власний глобальний обробник помилок — перехоплює всі необроблені помилки
@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  private logger = inject(LoggingService);

  handleError(error: unknown): void {
    const err = error instanceof Error ? error : new Error(String(error));
    this.logger.captureException(err);
    console.error('[GlobalError]', err);
  }
}

providers: [{ provide: ErrorHandler, useClass: GlobalErrorHandler }]

// HTTP interceptor — логування кожного запиту/відповіді з таймінгом
@Injectable()
export class LoggingInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<unknown>, next: HttpHandler): Observable<HttpEvent<unknown>> {
    const start = Date.now();
    return next.handle(req).pipe(
      tap(event => {
        if (event instanceof HttpResponse)
          console.log(`[${req.method}] ${req.url} → ${event.status} (${Date.now() - start}ms)`);
      }),
      catchError(err => {
        console.error(`[${req.method}] ${req.url} FAILED`, err);
        return throwError(() => err);
      })
    );
  }
}

Web Vitals та порогові значення Core Web Vitals

▶
// Вимірювання Core Web Vitals у Angular-застосунку
import { onCLS, onLCP, onINP, onFCP, onTTFB } from 'web-vitals';

function sendToAnalytics({ name, value, id }: Metric) {
  navigator.sendBeacon('/analytics', JSON.stringify({ name, value, id }));
}

onCLS(sendToAnalytics);   // Cumulative Layout Shift
onLCP(sendToAnalytics);   // Largest Contentful Paint
onINP(sendToAnalytics);   // Interaction to Next Paint
onFCP(sendToAnalytics);   // First Contentful Paint
onTTFB(sendToAnalytics);  // Time to First Byte

// Відстеження таймінгу навігації на кожному маршруті
@Injectable({ providedIn: 'root' })
export class RouteTimingService {
  constructor(private router: Router) {
    let navStart = 0;
    router.events.pipe(
      filter(e => e instanceof NavigationStart || e instanceof NavigationEnd)
    ).subscribe(e => {
      if (e instanceof NavigationStart) navStart = Date.now();
      if (e instanceof NavigationEnd)
        console.log(`Route ${e.url} loaded in ${Date.now() - navStart}ms`);
    });
  }
}
