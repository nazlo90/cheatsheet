Storage APIs

▶

WebSocket API та життєвий цикл

▶
// Життєвий цикл з'єднання
const ws = new WebSocket('wss://api.example.com/ws');

ws.onopen    = () => ws.send(JSON.stringify({ type: 'auth', token }));
ws.onmessage = (e) => { const data = JSON.parse(e.data); handle(data); };
ws.onclose   = (e) => console.log(`Closed: ${e.code} ${e.reason}`);
ws.onerror   = (e) => console.error('WS Error', e);

ws.send('text');
ws.send(JSON.stringify({ type: 'message', body: 'hello' }));
ws.close(1000, 'Normal closure');

// ReadyState
// CONNECTING=0, OPEN=1, CLOSING=2, CLOSED=3
ws.readyState === WebSocket.OPEN;

Патерн авто-перепідключення + RxJS WebSocket

▶
// Перепідключення з експоненціальним відступом
class ReconnectingWS {
  private ws: WebSocket | null = null;
  private retries = 0;

  connect() {
    this.ws = new WebSocket(this.url);
    this.ws.onclose = () => {
      const delay = Math.min(1000 * 2 ** this.retries++, 30000);
      setTimeout(() => this.connect(), delay);
    };
  }
}

// RxJS webSocket
import { webSocket } from 'rxjs/webSocket';
const subject = webSocket('wss://api.example.com/ws');
subject.subscribe(msg => handle(msg));
subject.next({ type: 'subscribe', channel: 'prices' });
subject.pipe(
  retryWhen(errors => errors.pipe(delay(2000)))
).subscribe();
В Angular завжди закривайте WebSocket у ngOnDestroy. Використовуйте shareReplay(1) для спільного використання одного з'єднання між кількома компонентами.

Service Workers

Концепція, життєвий цикл та стратегії кешування

▶

Окремий потік, що виступає мережевим проксі. Лише HTTPS. Забезпечує офлайн-режим, фонову синхронізацію, push-сповіщення.

Життєвий цикл: Installing → Waiting → Activating → Activated → Redundant

navigator.serviceWorker.register('/sw.js');
self.addEventListener('install', e => e.waitUntil(caches.open('v1').then(c => c.addAll(['/']))));
self.addEventListener('fetch', e => e.respondWith(
  caches.match(e.request).then(cached => cached || fetch(e.request))
));
СтратегіяКоли використовувати
Cache firstСтатичні ресурси
Network firstДані API — критично важлива свіжість
Stale-while-revalidateБаланс: віддати кеш, оновити у фоні
Network onlyЗапити автентифікації
Web Workers

Web Workers vs Service Workers

▶

Web Workers запускають скрипти у фонових потоках, окремо від головного UI-потоку — зберігаючи сторінку чутливою під час важких CPU-задач. Немає доступу до DOM; спілкуються через postMessage.

Web WorkerService Worker
ПризначенняCPU-обчисленняМережевий проксі / офлайн
Життєвий циклПрив'язаний до сторінкиНезалежний
DOMНемає доступуНемає доступу
const w = new Worker('worker.js');
w.postMessage({ data: largeArr });
w.onmessage = e => console.log(e.data);
// worker.js
self.onmessage = e => self.postMessage(heavyWork(e.data.data));
Angular 18 додав офіційну підтримку Web Workers. Використовуйте для: сортування/фільтрації, шифрування, обробки зображень — будь-чого, що блокує більше 16 мс.
