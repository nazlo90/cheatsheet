Архітектура Electron Основне

Головний процес проти процесу рендерера

▶

Electron запускає два типи процесів. Головний процес — це Node.js: він керує життєвим циклом застосунку та доступом до нативного ОС. Кожен процес рендерера — це веб-сторінка Chromium: він виконує код інтерфейсу і за замовчуванням ізольований.

Головний процесПроцес рендерера
Середовище виконанняNode.jsChromium (браузер)
КількістьОдин на застосунокОдин на BrowserWindow
Доступ до NodeПовнийВідсутній (ізольований)
Доступ до DOMВідсутнійПовний
Файл входуmain.jsindex.html
// main.js — Головний процес
const { app, BrowserWindow } = require('electron');

app.whenReady().then(() => {
  const win = new BrowserWindow({
    width: 1200, height: 800,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,   // обов'язково для безпеки
      sandbox: true,
    },
  });
  win.loadFile('index.html');
});
Ніколи не встановлюйте nodeIntegration: true у рендерері — це відкриває повний API Node для потенційно ненадійного веб-вмісту.

Preload-скрипт і contextBridge

▶

Preload-скрипт запускається в рендерері до будь-якого веб-вмісту, маючи доступ до обмеженого набору API Node/Electron. contextBridge безпечно відкриває контрольований API рендереру.

// preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('electronAPI', {
  openFile: () => ipcRenderer.invoke('dialog:openFile'),
  onUpdate: (cb) => ipcRenderer.on('update-available', cb),
  removeUpdate: () => ipcRenderer.removeAllListeners('update-available'),
});

// рендерер — window.electronAPI тепер доступний
const filePath = await window.electronAPI.openFile();
Ніколи не передавайте сирий об'єкт ipcRenderer через contextBridge — це дозволить рендереру надсилати довільні IPC-канали. Відкривайте лише іменовані конкретні функції.
IPC-комунікація IPC

Патерни ipcMain / ipcRenderer

▶

Міжпроцесна комунікація (IPC) — єдиний спосіб обміну даними між процесами рендерера і головним процесом. Використовуйте invoke/handle для запит-відповідь і send/on для «надіслав і забув».

ПатернЗ боку рендерераЗ боку головного процесуКоли використовувати
Запит / ВідповідьipcRenderer.invoke()ipcMain.handle()Потрібна відповідь (async)
Одностороннє (R→M)ipcRenderer.send()ipcMain.on()Надіслати і забути до головного
Одностороннє (M→R)ipcRenderer.on()win.webContents.send()Push з головного до рендерера
// main.js
ipcMain.handle('dialog:openFile', async () => {
  const { canceled, filePaths } = await dialog.showOpenDialog({});
  if (canceled) return null;
  return filePaths[0];
});

// Push-сповіщення з головного → рендерер
autoUpdater.on('update-available', () => {
  win.webContents.send('update-available');
});
Валідуйте всі дані, отримані в ipcMain.handle — скомпрометований рендерер може надсилати довільні навантаження.

Іменування IPC-каналів і валідація

▶
// Використовуйте білий список дозволених каналів у preload
const ALLOWED_CHANNELS = ['app:getVersion', 'file:read', 'file:write'];

contextBridge.exposeInMainWorld('ipc', {
  invoke: (channel, ...args) => {
    if (!ALLOWED_CHANNELS.includes(channel)) throw new Error('Blocked channel');
    return ipcRenderer.invoke(channel, ...args);
  },
});

// main.js — завжди валідуйте аргументи
ipcMain.handle('file:read', async (event, filePath) => {
  if (typeof filePath !== 'string') throw new Error('Invalid path');
  const safe = path.resolve(filePath);
  if (!safe.startsWith(ALLOWED_DIR)) throw new Error('Path traversal denied');
  return fs.promises.readFile(safe, 'utf8');
});

Параметри BrowserWindow

▶
const win = new BrowserWindow({
  width: 1200, height: 800,
  minWidth: 600, minHeight: 400,
  frame: false,          // без рамки (власний заголовок)
  titleBarStyle: 'hidden',  // лише кнопки macOS
  show: false,           // чекати до ready-to-show
  backgroundColor: '#1a1a1a',
  webPreferences: {
    preload: path.join(__dirname, 'preload.js'),
    contextIsolation: true,
    sandbox: true,
    spellcheck: false,
  },
});

win.once('ready-to-show', () => win.show());  // уникнути білого спалаху
win.loadFile('index.html');
Завжди встановлюйте show: false і показуйте вікно на ready-to-show, щоб уникнути білого спалаху при запуску.

Діалоги, меню і системний трей

▶
// Діалог
const { dialog, Menu, Tray, nativeImage } = require('electron');

const { filePaths } = await dialog.showOpenDialog(win, {
  title: 'Open File',
  filters: [{ name: 'JSON', extensions: ['json'] }],
  properties: ['openFile'],

// Меню застосунку
const menu = Menu.buildFromTemplate([
  { label: 'File', submenu: [
    { label: 'Open', accelerator: 'CmdOrCtrl+O', click: openFile },
    { type: 'separator' },
    { role: 'quit' },
  ]},
]);
Menu.setApplicationMenu(menu);

// Системний трей
const tray = new Tray(nativeImage.createFromPath('icon.png'));
tray.setContextMenu(Menu.buildFromTemplate([
  { label: 'Show', click: () => win.show() },
  { label: 'Quit', click: () => app.quit() },
]));

Сповіщення і shell

▶
const { Notification, shell, clipboard } = require('electron');

// Сповіщення ОС
new Notification({ title: 'Done', body: 'Export complete' }).show();

// Відкрити URL у браузері за замовчуванням (безпечніше за loadURL)
await shell.openExternal('https://example.com');

// Відкрити файл у стандартному застосунку ОС
await shell.openPath('/path/to/file.pdf');

// Буфер обміну
clipboard.writeText('copied!');
const text = clipboard.readText();
Використовуйте shell.openExternal лише з перевіреними URL — передача ненадійного введення користувача може виконати довільні програми (наприклад, шляхи file://).
Пакування і розповсюдження Build

Конфігурація electron-builder

▶

electron-builder — стандартний інструмент для пакування Electron-застосунків. Він створює нативні платформові інсталятори (.dmg, .exe, .AppImage) і обробляє підпис коду.

// package.json
{
  "build": {
    "appId": "com.company.myapp",
    "productName": "MyApp",
    "directories": { "output": "dist" },
    "files": ["dist/**", "main.js", "preload.js"],
    "mac": {
      "category": "public.app-category.productivity",
      "hardenedRuntime": true,
      "entitlements": "entitlements.mac.plist"
    },
    "win": { "target": ["nsis"] },
    "linux": { "target": ["AppImage", "deb"] },
    "publish": { "provider": "github" }
  }
}

Автооновлення з electron-updater

▶
const { autoUpdater } = require('electron-updater');

autoUpdater.checkForUpdatesAndNotify();

// Або ручний сценарій
autoUpdater.on('update-available', () => {
  win.webContents.send('update-available');
});
autoUpdater.on('update-downloaded', () => {
  win.webContents.send('update-downloaded');
});

// Запускається з рендерера через IPC
ipcMain.on('install-update', () => autoUpdater.quitAndInstall());
Вкажіть "publish": { "provider": "github" } у конфігурації electron-builder і опублікуйте GitHub Release — electron-updater виявить його автоматично.
